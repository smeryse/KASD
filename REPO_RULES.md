# Repository Structure and Naming Rules

## 1. Общие принципы

* Репозиторий должен иметь **предсказуемую и единообразную структуру**.
* Все директории и файлы должны быть логически сгруппированы по назначению.
* Одна сущность → одно место хранения.
* Исключается смешивание:

  * исходного кода
  * учебных материалов
  * сборочных артефактов
  * заметок
  * ресурсов

Структура должна быть масштабируемой и удобной для навигации.

---

## 2. Структура репозитория (обязательная)

Корневая структура:

```
.
├── docs/          # лекции, теория, литература, шпоры
├── labs/          # лабораторные работы
├── tasks/         # отдельные задачи и реализации алгоритмов
├── contests/      # контесты и решения
├── exams/         # подготовка к экзаменам
├── materials/     # дополнительные материалы курса
├── assets/        # изображения и ресурсы
├── meta/          # информация о курсе, структура курса
├── README.md
└── TODO.md
```

---

## 3. Структура docs/

```
docs/
├── lectures/      # лекционные материалы
├── theory/        # теоретические конспекты
├── literature/    # книги и справочные материалы
└── cheatsheets/   # шпаргалки
```

### Требования

* Лекции — только материалы лекций.
* Теория — обобщённые знания и конспекты.
* Не смешивать PDF лекций и теоретические заметки.

---

## 4. Структура лабораторных работ

```
labs/
├── lab01-topic/
│   ├── src/
│   ├── report.pdf
│   └── README.md
```

### Требования

* Каждая лабораторная — отдельная директория.
* Используется единый шаблон структуры.
* Названия лабораторных — `labNN-topic`.

---

## 5. Структура задач

```
tasks/
├── task-name/
│   ├── src/
│   ├── tests/
│   └── README.md
```

### Требования

* Одна задача = одна директория.
* Код и тесты разделены.
* Названия описывают задачу.

---

## 6. Структура контестов

```
contests/
├── ct01/
│   ├── src/
│   ├── tasks/
│   ├── solutions/
│   └── explanations/
```

### Требования

* Контесты нумеруются.
* Решения и условия разделены.

---

## 7. Структура экзаменационных материалов

```
exams/
├── theory/
├── tasks/
├── answers/
└── prep/
```

---

## 8. Assets (ресурсы)

```
assets/
├── diagrams/
├── screenshots/
└── figures/
```

### Требования

* Только используемые ресурсы.
* Запрещены временные файлы и вставки вида `Pasted image`.

---

## 9. Правила именования

### 9.1 Язык

* Названия директорий — английский.
* Содержимое файлов допускается на русском.

---

### 9.2 Стиль имен

Используется **kebab-case**:

```
dynamic-programming
vector-norm
priority-queue
```

Запрещено:

```
VectorNorm
vectorNorm
Vector_Norm
```

---

### 9.3 Нумерация

При необходимости используется формат:

```
01-topic
02-topic
```

Нумерация фиксирует порядок.

---

## 10. Build-файлы и временные файлы

Следующие файлы запрещены в репозитории:

```
bin/
obj/
*.cache
*.user
```

Они должны быть добавлены в `.gitignore`.

---

## 11. README

Каждый крупный раздел должен содержать `README.md` с:

* описанием содержимого
* структурой директории
* назначением материалов

---

## 12. Недопустимые практики

Запрещается:

* хранение сборочных артефактов
* дублирование материалов
* смешение типов контента
* хаотичное именование
* использование временных файлов
* отсутствие структуры

---

## 13. Цель структуры

Структура должна обеспечивать:

* быструю навигацию
* масштабируемость
* предсказуемость
* единый стандарт хранения материалов
* удобство поддержки
