# ✅ **2. Инфологическая модель данных (ER-модель)**

## 🔗 Связи между сущностями

```
Artist 1 ─── N Track
Genre 1 ─── N Track
Playlist N ─── M Track (через PlaylistTrack)
Set N ─── M Track (через SetTrack)
Event 1 ─── 1 Set
```

## 📌 Типы связей

|Связь|Тип|
|---|---|
|Artist — Track|1:N|
|Genre — Track|1:N|
|Playlist — Track|N:M|
|Set — Track|N:M|
|Event — Set|1:1|

---

# ✅ **3. Входные и выходные потоки информации**

## 📥 3.1. Входные потоки

|Источник|Данные|Форма|Частота|Трудоёмкость|
|---|---|---|---|---|
|DJ|Метаданные треков|Ручной ввод|При добавлении|Средняя|
|Аудиофайлы|Треки|Файлы (MP3/WAV)|Нерегулярно|Высокая|
|DJ|Плейлисты|GUI-формы|Часто|Низкая|
|DJ|Сеты|GUI-формы|Перед выступлением|Средняя|
|DJ|История выступлений|Форма отчёта|После выступления|Низкая|

---

## 📤 3.2. Выходные потоки

|Получатель|Данные|Форма|Частота|
|---|---|---|---|
|DJ|Список треков|Таблица/GUI|Постоянно|
|DJ|Готовый сет|Экспорт|Перед выступлением|
|DJ|История сетов|Отчёт|По запросу|
|DJ|Аналитика (жанры, BPM)|Диаграммы|По запросу|

---

## 🔄 3.3. Блок-схема инфопотоков (текстово)

```
[DJ] → (Ввод треков) → [База треков]
[DJ] → (Создание плейлистов) → [Плейлисты]
[DJ] → (Формирование сетов) → [Сеты]
[DJ] → (Выступление) → [История выступлений]
[Система] → (Отчёты, аналитика) → [DJ]
```

---

# ✅ **4. Процедуры преобразования потоков**

## 🔧 Основные процедуры

|Процедура|Вход|Выход|
|---|---|---|
|Добавление трека|Аудиофайл + метаданные|Запись в Track|
|Классификация|Track|Связь с Genre|
|Формирование плейлиста|Track|Playlist|
|Подготовка сета|Playlist|Set|
|Фиксация выступления|Set + Event данные|Event|
|Анализ истории|Event|Отчёты|
