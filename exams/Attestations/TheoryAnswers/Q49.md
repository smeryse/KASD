### 49. Интерфейс vs абстрактный класс; сколько интерфейсов; наследование интерфейсов; реализация; модификаторы
Интерфейс — чистый контракт (в классическом виде без реализации/состояния); абстрактный класс может содержать поля и готовые методы. Класс может реализовать множество интерфейсов. Интерфейсы могут наследоваться. Все члены интерфейса публичные по умолчанию, реализовать нужно все (кроме default‑методов в новых версиях).  
```csharp
interface IFly{ void Fly(); } class Bird:IFly{ public void Fly(){} }
```
