### 3. Сортировка слиянием. Почему всегда `O(n log n)`. Мастер‑теорема. Применение к merge и бинарному поиску
Merge sort: делим пополам, рекурсивно сортируем, линейно сливаем. На уровне рекурсии просматриваем все n элементов (`O(n)`), а глубина деления на 2 — `log n`, поэтому суммарно `O(n log n)` независимо от входа (нет “плохих” случаев).  
Мастер‑теорема для `T(n)=aT(n/b)+f(n)`: сравниваем `f(n)` с `n^{log_b a}`. Если f меньше — ответ `Θ(n^{log_b a})`; если равен — `Θ(n^{log_b a} log n)`; если больше (и регулярность) — `Θ(f(n))`. Для merge (`a=2,b=2,f=Θ(n)`) — второй случай ⇒ `Θ(n log n)`. Для бинарного поиска `T(n)=T(n/2)+O(1)` — первый/второй случай с `a=1,b=2,f=Θ(1)` ⇒ `Θ(log n)`.  
```csharp
void MergeSort(int[] a,int l,int r,int[] tmp){
    if(l>=r) return;
    int m=(l+r)/2;
    MergeSort(a,l,m,tmp); MergeSort(a,m+1,r,tmp);
    int i=l,j=m+1,k=l;
    while(i<=m && j<=r) tmp[k++]= a[i]<=a[j]? a[i++]:a[j++];
    while(i<=m) tmp[k++]=a[i++]; while(j<=r) tmp[k++]=a[j++];
    Array.Copy(tmp,l,a,l,r-l+1);
}
```
