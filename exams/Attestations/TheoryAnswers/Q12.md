### 12. DSU: операции; применения; наив и его недостатки
Вопрос из pdf: “Какие две основные операции поддерживает система непересекающихся множеств? Где она применяется? Какова сложность наивной реализации?”  
Операции: `Find` возвращает представителя множества; `Union` объединяет два множества. Применения: MST Крускала, онлайн-объединения компонент, динамическая связность. Наив: родительский массив без эвристик, высота дерева может вырасти до `O(n)`, тогда `Find/Union` работают за линейное время.  
```csharp
class DisjointSet
{
    private readonly int[] parent;

    public DisjointSet(int n)
    {
        parent = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    public int Find(int v)
    {
        if (parent[v] == v) return v;
        // без сжатия путей это была бы простая рекурсия: return Find(parent[v]);
        parent[v] = Find(parent[v]); // добавили сжатие для ускорения
        return parent[v];
    }

    public void Union(int a, int b)
    {
        a = Find(a);
        b = Find(b);
        if (a == b) return;
        parent[b] = a; // наивно — всегда подвешиваем одно под другое
    }
}
```
Без эвристик (сжатия путей/рангов) дерево может стать цепочкой ⇒ `O(n)` на `Find/Union`; эвристики дают амортизированное `O(α(n))` (см. п.13).
