### 1. Строки и хеширование (полиномиальные хеши)

**Условие:** Даны две строки: текст $s$ длины $n$ и образец $t$ длины $m$ $(m \le n)$. Требуется найти все позиции в тексте, где образец входит с точностью до замены не более одного символа.

**Решение:**

Для решения этой задачи будем использовать полиномиальные хеши, которые позволяют сравнивать подстроки за $O(1)$ после предварительной обработки. Основная идея заключается в том, что если две подстроки отличаются не более чем одним символом, то можно проверить это, сравнив хеши префиксов и суффиксов вокруг позиции возможного различия.

**Полиномиальный хеш** определяется следующим образом: для строки $s$ длины $n$ хеш вычисляется как значение многочлена в точке $x$ по модулю $p$:

$$\text{hash}(s) = (s[0] \cdot x^{n-1} + s[1] \cdot x^{n-2} + \ldots + s[n-1] \cdot x^0) \mod p$$

где $x$ — основание (обычно выбирается случайным или фиксированным, например, 31 или 37), $p$ — большое простое число (часто $10^9 + 7$ или $10^9 + 9$).

**Предобработка:**

1. Предвычислим степени $x^i \mod p$ для всех $i$ от 0 до $n$.
2. Вычислим префиксные хеши для строки $s$: $\text{pref\_hash}[i]$ — хеш префикса $s[0:i]$.
3. Вычислим префиксные хеши для строки $t$ аналогично.

Теперь хеш любой подстроки $s[l:r]$ можно получить за $O(1)$:

$$\text{hash}(s[l:r]) = (\text{pref\_hash}[r] - \text{pref\_hash}[l] \cdot x^{r-l}) \mod p$$

**Алгоритм проверки позиции:**

Для каждой позиции $i$ в тексте $s$, где может начинаться вхождение образца (то есть $i$ от 0 до $n-m$), нужно проверить, отличается ли подстрока $s[i:i+m]$ от образца $t$ не более чем в одном символе.

Проверку можно выполнить за $O(1)$ с использованием хешей и бинарного поиска:

1. Сначала сравним полные хеши $s[i:i+m]$ и $t$. Если они равны, то подстроки идентичны (с высокой вероятностью), и различий нет.
2. Если хеши не равны, найдём позицию первого различия с помощью бинарного поиска. Для этого будем проверять равенство префиксов возрастающей длины: если $\text{hash}(s[i:i+k]) = \text{hash}(t[0:k])$, то первые $k$ символов совпадают.
3. После нахождения позиции первого различия $k$, сравним хеши оставшихся суффиксов: $\text{hash}(s[i+k+1:i+m])$ и $\text{hash}(t[k+1:m])$. Если они равны, то различие ровно одно.

**Псевдокод:**

```csharp
public static List<int> FindWithOneMismatch(string s, string t, int p = 1_000_000_007, int x = 31)
{
    int n = s.Length, m = t.Length;
    var result = new List<int>();
    
    // Предвычисление степеней
    long[] pows = new long[n + 1];
    pows[0] = 1;
    for (int i = 1; i <= n; i++)
        pows[i] = (pows[i - 1] * x) % p;
    
    // Префиксные хеши для s
    long[] prefS = new long[n + 1];
    for (int i = 0; i < n; i++)
        prefS[i + 1] = (prefS[i] * x + s[i]) % p;
    
    // Префиксные хеши для t
    long[] prefT = new long[m + 1];
    for (int i = 0; i < m; i++)
        prefT[i + 1] = (prefT[i] * x + t[i]) % p;
    
    // Функция получения хеша подстроки
    long GetHash(long[] pref, int l, int r, long[] pows)
    {
        long h = (pref[r] - pref[l] * pows[r - l]) % p;
        return h < 0 ? h + p : h;
    }
    
    for (int i = 0; i <= n - m; i++)
    {
        // Сравниваем полные хеши
        if (GetHash(prefS, i, i + m, pows) == GetHash(prefT, 0, m, pows))
        {
            result.Add(i); // Полное совпадение
            continue;
        }
        
        // Бинарный поиск позиции первого различия
        int left = 0, right = m;
        while (left < right)
        {
            int mid = (left + right) / 2;
            if (GetHash(prefS, i, i + mid, pows) == GetHash(prefT, 0, mid, pows))
                left = mid + 1;
            else
                right = mid;
        }
        
        // left - позиция первого различия
        // Проверяем, совпадают ли суффиксы после различия
        if (left < m && GetHash(prefS, i + left + 1, i + m, pows) == GetHash(prefT, left + 1, m, pows))
            result.Add(i); // Ровно одно различие
    }
    
    return result;
}
```

**Оценка сложности:**

- Предобработка: $O(n + m)$ на вычисление степеней и префиксных хешей.
- Проверка каждой позиции: $O(\log m)$ на бинарный поиск.
- Общее время: $O(n \log m)$.

Можно улучшить до $O(n)$, если использовать более хитрый подход с поиском первого и последнего различия, но $O(n \log m)$ вполне достаточно для большинства задач.

**Важно:** Для надёжности рекомендуется использовать двойное хеширование — два разных модуля $p_1$ и $p_2$, чтобы минимизировать вероятность коллизий.

### 2. Префикс-функция и поиск с циклическим сдвигом

**Условие:** Строки $s$ и $t$ считаются циклически эквивалентными, если существует число $k$ (сдвиг по модулю алфавита), такое что каждый символ $t[j]$ получается из $s[j]$ прибавлением $k$. Требуется найти все вхождения образца $p$ в текст $t$ с точностью до такого циклического сдвига.

**Решение:**

Данная задача является модификацией классического поиска подстроки, но с дополнительным условием: мы ищем не точное совпадение, а совпадение с точностью до постоянного сдвига всех символов. Например, строки "abc" и "def" циклически эквивалентны при $k = 3$ (если считать 'a' = 0, 'b' = 1, и т.д.).

**Ключевая идея:**

Вместо работы с самими символами, будем работать с **разностями соседних символов**. Если две строки циклически эквивалентны, то последовательности разностей их соседних символов совпадают. Действительно, если $t[i] = s[i] + k$ для всех $i$, то:

$$t[i+1] - t[i] = (s[i+1] + k) - (s[i] + k) = s[i+1] - s[i]$$

Таким образом, задача сводится к поиску подстроки в массиве разностей.

**Алгоритм:**

1. Построим массив разностей для образца $p$: $\text{diff}_p[i] = p[i+1] - p[i]$ для $i = 0, \ldots, |p|-2$.
2. Построим массив разностей для текста $t$: $\text{diff}_t[i] = t[i+1] - t[i]$ для $i = 0, \ldots, |t|-2$.
3. Используем алгоритм Кнута-Морриса-Пратта (КМП) для поиска $\text{diff}_p$ в $\text{diff}_t$.
4. Каждое найденное вхождение на позиции $i$ в массиве разностей соответствует потенциальному вхождению образца в тексте на позиции $i$.

**Модификация префикс-функции:**

Префикс-функция $\pi[i]$ для массива $a$ определяется как длина наибольшего собственного префикса $a[0:\pi[i]]$, который является суффиксом $a[0:i+1]$. Алгоритм вычисления остаётся тем же, но работаем мы с массивом разностей, а не с исходными символами.

**Псевдокод:**

```csharp
public static List<int> FindCyclicShifts(string text, string pattern)
{
    var result = new List<int>();
    
    if (pattern.Length > text.Length)
        return result;
    
    if (pattern.Length == 1)
    {
        // Если образец из одного символа, подходит любая позиция
        for (int i = 0; i < text.Length; i++)
            result.Add(i);
        return result;
    }
    
    // Строим массивы разностей
    int[] diffPattern = new int[pattern.Length - 1];
    for (int i = 0; i < pattern.Length - 1; i++)
        diffPattern[i] = pattern[i + 1] - pattern[i];
    
    int[] diffText = new int[text.Length - 1];
    for (int i = 0; i < text.Length - 1; i++)
        diffText[i] = text[i + 1] - text[i];
    
    // Вычисляем префикс-функцию для образца
    int[] pi = ComputePrefixFunction(diffPattern);
    
    // Поиск КМП
    int j = 0; // индекс в образце
    for (int i = 0; i < diffText.Length; i++)
    {
        while (j > 0 && diffText[i] != diffPattern[j])
            j = pi[j - 1];
        
        if (diffText[i] == diffPattern[j])
            j++;
        
        if (j == diffPattern.Length)
        {
            // Нашли вхождение, начинающееся на позиции i - j + 1 в массиве разностей
            // что соответствует позиции i - j + 1 в исходном тексте
            result.Add(i - j + 1);
            j = pi[j - 1];
        }
    }
    
    return result;
}

private static int[] ComputePrefixFunction(int[] arr)
{
    int n = arr.Length;
    int[] pi = new int[n];
    
    for (int i = 1; i < n; i++)
    {
        int j = pi[i - 1];
        while (j > 0 && arr[i] != arr[j])
            j = pi[j - 1];
        
        if (arr[i] == arr[j])
            j++;
        
        pi[i] = j;
    }
    
    return pi;
}
```

**Обоснование корректности:**

Если $\text{diff}_p$ совпадает с подмассивом $\text{diff}_t$, начиная с позиции $i$, то для всех $k$ выполняется $p[k+1] - p[k] = t[i+k+1] - t[i+k]$. Это означает, что разности соседних символов совпадают, а значит, существует константа $k$, такая что $t[i+m] = p[m] + k$ для всех $m$.

**Оценка сложности:**

- Построение массивов разностей: $O(|t| + |p|)$.
- Вычисление префикс-функции: $O(|p|)$.
- Поиск КМП: $O(|t|)$.
- **Итого:** $O(|t| + |p|)$ — линейное время.

**Пример:**

Пусть текст $t = \text{"xyzabcde"}$, образец $p = \text{"abc"}$.

- $\text{diff}_p = [1, 1]$ (так как 'b' - 'a' = 1, 'c' - 'b' = 1).
- $\text{diff}_t = [1, 1, -23, 1, 1, 1, 1]$.

КМП найдёт вхождение $[1, 1]$ на позиции 3 в $\text{diff}_t$, что соответствует позиции 3 в тексте (подстрока "abc").

### 3. Алгоритм Рабина-Карпа и поиск повторяющихся подстрок

**Условие:** Дана строка $s$ длины $n$. Найдите самую длинную подстроку, которая встречается в $s$ не менее двух раз (пересечения разрешены).

**Решение:**

Эта задача решается комбинацией **бинарного поиска по длине подстроки** и **хеширования** для проверки существования повторяющейся подстроки заданной длины.

**Основная идея:**

Если существует подстрока длины $L$, встречающаяся дважды, то существует и подстрока длины $L-1$, встречающаяся дважды (просто возьмём префикс длины $L-1$). Это свойство монотонности позволяет использовать бинарный поиск по длине.

**Алгоритм:**

1. **Бинарный поиск по длине:** Ищем максимальную длину $L$, для которой существует повторяющаяся подстрока. Диапазон поиска: от 1 до $n-1$.
2. **Проверка длины $L$:** Для фиксированной длины $L$ вычислим хеши всех подстрок длины $L$ и сохраним их в хеш-таблицу. Если какой-то хеш встречается дважды, значит, найдена повторяющаяся подстрока.
3. **Избежание коллизий:** Используем двойное хеширование с двумя разными модулями $p_1$ и $p_2$ и основаниями $x_1$ и $x_2$. Это делает вероятность коллизии пренебрежимо малой.

**Псевдокод:**

```csharp
public static string FindLongestRepeatedSubstring(string s)
{
    int n = s.Length;
    if (n == 0) return "";
    
    // Параметры для двойного хеширования
    int p1 = 1_000_000_007, x1 = 31;
    int p2 = 1_000_000_009, x2 = 37;
    
    // Предвычисление степеней
    long[] pows1 = new long[n + 1], pows2 = new long[n + 1];
    pows1[0] = 1;
    pows2[0] = 1;
    for (int i = 1; i <= n; i++)
    {
        pows1[i] = (pows1[i - 1] * x1) % p1;
        pows2[i] = (pows2[i - 1] * x2) % p2;
    }
    
    // Префиксные хеши
    long[] pref1 = new long[n + 1], pref2 = new long[n + 1];
    for (int i = 0; i < n; i++)
    {
        pref1[i + 1] = (pref1[i] * x1 + s[i]) % p1;
        pref2[i + 1] = (pref2[i] * x2 + s[i]) % p2;
    }
    
    long GetHash1(int l, int r)
    {
        long h = (pref1[r] - pref1[l] * pows1[r - l]) % p1;
        return h < 0 ? h + p1 : h;
    }
    
    long GetHash2(int l, int r)
    {
        long h = (pref2[r] - pref2[l] * pows2[r - l]) % p2;
        return h < 0 ? h + p2 : h;
    }
    
    // Проверка существования повторяющейся подстроки длины len
    bool HasRepeat(int len)
    {
        var seen = new HashSet<(long, long)>();
        for (int i = 0; i <= n - len; i++)
        {
            var hash = (GetHash1(i, i + len), GetHash2(i, i + len));
            if (seen.Contains(hash))
                return true;
            seen.Add(hash);
        }
        return false;
    }
    
    // Бинарный поиск максимальной длины
    int left = 1, right = n - 1, bestLen = 0;
    while (left <= right)
    {
        int mid = (left + right) / 2;
        if (HasRepeat(mid))
        {
            bestLen = mid;
            left = mid + 1;
        }
        else
        {
            right = mid - 1;
        }
    }
    
    if (bestLen == 0) return "";
    
    // Находим саму подстроку
    var seen = new HashSet<(long, long)>();
    for (int i = 0; i <= n - bestLen; i++)
    {
        var hash = (GetHash1(i, i + bestLen), GetHash2(i, i + bestLen));
        if (seen.Contains(hash))
            return s.Substring(i, bestLen);
        seen.Add(hash);
    }
    
    return "";
}
```

**Оценка сложности:**

- Предобработка: $O(n)$.
- Бинарный поиск: $O(\log n)$ итераций.
- Проверка длины: $O(n)$ на каждой итерации.
- **Итого:** $O(n \log n)$.

**Как избежать коллизий:**

1. **Двойное хеширование:** Используем два независимых хеша с разными параметрами. Вероятность одновременной коллизии обоих хешей крайне мала.
2. **Большие простые модули:** Выбираем $p \approx 10^9$, что даёт вероятность коллизии порядка $1/p$.
3. **Случайное основание:** Можно выбирать $x$ случайным образом на каждом запуске.

Для абсолютной гарантии можно после нахождения кандидата с совпадающими хешами выполнить прямое сравнение подстрок, но это увеличит сложность в худшем случае.

### 4. Z-функция и периоды строки

**Условие:** Дана строка $s$ длины $n$. Периодом строки называется такое целое $k$ $(1 \le k \le n - 1)$, что для всех $i$ от 0 до $n - k - 1$ выполняется $s[i] = s[i + k]$. Как с помощью Z-функции найти все периоды строки за $O(n)$?

**Решение:**

**Z-функция** для строки $s$ — это массив $z$ длины $n$, где $z[i]$ равен длине наибольшего общего префикса строки $s$ и её суффикса, начинающегося в позиции $i$. Формально:

$$z[i] = \max\{k : s[0:k] = s[i:i+k]\}$$

По определению, $z[0]$ обычно не определён или считается равным 0 (так как это тривиальный случай).

**Связь Z-функции с периодами:**

Число $k$ является периодом строки $s$ тогда и только тогда, когда $z[k] = n - k$. Действительно:
- Если $k$ — период, то $s[i] = s[i+k]$ для всех допустимых $i$, то есть суффикс, начинающийся в позиции $k$, совпадает с префиксом длины $n-k$. Значит, $z[k] \ge n - k$. Но $z[k]$ не может быть больше $n - k$ (длина суффикса), поэтому $z[k] = n - k$.
- Обратно, если $z[k] = n - k$, то суффикс длины $n-k$, начинающийся в позиции $k$, совпадает с префиксом той же длины, что в точности означает условие периодичности.

**Алгоритм:**

1. Вычислим Z-функцию для строки $s$ за $O(n)$.
2. Пройдём по всем $k$ от 1 до $n-1$ и проверим условие $z[k] = n - k$.
3. Все $k$, удовлетворяющие условию, являются периодами.

**Псевдокод:**

```csharp
public static List<int> FindAllPeriods(string s)
{
    int n = s.Length;
    var periods = new List<int>();
    
    int[] z = ComputeZFunction(s);
    
    for (int k = 1; k < n; k++)
    {
        if (z[k] == n - k)
            periods.Add(k);
    }
    
    return periods;
}

private static int[] ComputeZFunction(string s)
{
    int n = s.Length;
    int[] z = new int[n];
    
    int l = 0, r = 0; // [l, r] - интервал совпадения с префиксом
    
    for (int i = 1; i < n; i++)
    {
        // Инициализируем z[i] используя ранее вычисленные значения
        if (i <= r)
            z[i] = Math.Min(r - i + 1, z[i - l]);
        
        // Наивное продолжение
        while (i + z[i] < n && s[z[i]] == s[i + z[i]])
            z[i]++;
        
        // Обновляем интервал, если зашли дальше
        if (i + z[i] - 1 > r)
        {
            l = i;
            r = i + z[i] - 1;
        }
    }
    
    return z;
}
```

**Пример:**

Рассмотрим строку $s = \text{"abababa"}$ длины $n = 7$.

Z-функция: $z = [0, 0, 5, 0, 3, 0, 1]$.

Проверяем периоды:
- $k = 1$: $z[1] = 0 \neq 6$ — не период.
- $k = 2$: $z[2] = 5 = 7 - 2$ — **период** (действительно, "ab" повторяется).
- $k = 3$: $z[3] = 0 \neq 4$ — не период.
- $k = 4$: $z[4] = 3 \neq 3$ — **период** (совпадает!).
- $k = 5$: $z[5] = 0 \neq 2$ — не период.
- $k = 6$: $z[6] = 1 \neq 1$ — **период**.

Периоды: 2, 4, 6.

**Оценка сложности:**

- Вычисление Z-функции: $O(n)$.
- Проверка всех $k$: $O(n)$.
- **Итого:** $O(n)$.

### 5. Бор и автомат Ахо-Корасик

**Условие:** Дан набор ключевых слов и длинный текст. Необходимо найти в тексте все вхождения ключевых слов, которые начинаются и заканчиваются на гласную букву. Можно ли адаптировать автомат Ахо-Корасик так, чтобы он сразу выдавал только такие вхождения?

**Решение:**

**Бор (префиксное дерево)** — это структура данных для хранения набора строк, где каждая вершина соответствует префиксу одной или нескольких строк. Путь от корня до вершины определяет строку, а метки в вершинах могут хранить дополнительную информацию (например, является ли вершина концом какого-то ключевого слова).

**Автомат Ахо-Корасик** строится на основе бора и добавляет **суффиксные ссылки**, которые позволяют эффективно переходить между вершинами при несовпадении символов. Суффиксная ссылка из вершины $u$ ведёт в вершину, соответствующую наибольшему собственному суффиксу строки, представленной вершиной $u$, который также является префиксом какого-то ключевого слова.

**Адаптация для фильтрации по гласным:**

Да, автомат можно адаптировать для немедленной выдачи только тех вхождений, которые начинаются и заканчиваются на гласную. Для этого нужно хранить в каждой вершине дополнительную информацию:

1. **Первый символ пути:** В каждой вершине запомним первый символ на пути от корня до этой вершины. Это можно сделать при построении бора: для детей корня первый символ — это метка ребра, для остальных — наследуется от родителя.
2. **Флаг окончания:** В вершинах, соответствующих концам ключевых слов, храним список завершающихся здесь шаблонов (или просто флаг, если важно только наличие).
3. **Проверка при обходе:** При обработке текста, когда мы попадаем в вершину автомата, проверяем:
   - Является ли первый символ пути гласным.
   - Является ли текущий символ текста (последний символ вхождения) гласным.
   - Является ли вершина конечной для какого-то ключевого слова.

**Структура данных вершины:**

```csharp
public class AhoCorasickNode
{
    public Dictionary<char, AhoCorasickNode> Children { get; } = new();
    public AhoCorasickNode? SuffixLink { get; set; }
    public bool IsEnd { get; set; } = false; // Является ли конец слова
    public char FirstChar { get; set; } // Первый символ пути
    public List<string> Words { get; } = new(); // Слова, заканчивающиеся здесь
}
```

**Построение автомата:**

```csharp
public class AhoCorasickAutomaton
{
    private readonly AhoCorasickNode _root;
    private readonly HashSet<char> _vowels;
    
    public AhoCorasickAutomaton(IEnumerable<string> patterns, string alphabet = "abcdefghijklmnopqrstuvwxyz")
    {
        _root = new AhoCorasickNode();
        _vowels = new HashSet<char> { 'a', 'e', 'i', 'o', 'u', 'y', 
                                       'а', 'е', 'ё', 'и', 'о', 'у', 'ы', 'э', 'ю', 'я' };
        BuildTrie(patterns);
        BuildSuffixLinks();
    }
    
    private void BuildTrie(IEnumerable<string> patterns)
    {
        foreach (var pattern in patterns)
        {
            if (string.IsNullOrEmpty(pattern)) continue;
            
            var node = _root;
            for (int i = 0; i < pattern.Length; i++)
            {
                char c = pattern[i];
                if (!node.Children.ContainsKey(c))
                {
                    node.Children[c] = new AhoCorasickNode
                    {
                        FirstChar = i == 0 ? c : node.FirstChar
                    };
                }
                node = node.Children[c];
            }
            node.IsEnd = true;
            node.Words.Add(pattern);
        }
    }
    
    private void BuildSuffixLinks()
    {
        var queue = new Queue<AhoCorasickNode>();
        
        // Суффиксные ссылки для детей корня
        foreach (var child in _root.Children.Values)
        {
            child.SuffixLink = _root;
            queue.Enqueue(child);
        }
        
        // BFS для построения остальных ссылок
        while (queue.Count > 0)
        {
            var node = queue.Dequeue();
            
            foreach (var (c, child) in node.Children)
            {
                var suffix = node.SuffixLink;
                while (suffix != null && !suffix.Children.ContainsKey(c))
                    suffix = suffix.SuffixLink;
                
                child.SuffixLink = suffix?.Children.GetValueOrDefault(c, _root) ?? _root;
                queue.Enqueue(child);
            }
        }
    }
    
    public List<(int Position, string Word)> FindAllMatches(string text)
    {
        var result = new List<(int, string)>();
        var node = _root;
        
        for (int i = 0; i < text.Length; i++)
        {
            char c = text[i];
            
            // Переход по автомату
            while (node != _root && !node.Children.ContainsKey(c))
                node = node.SuffixLink!;
            
            if (node.Children.ContainsKey(c))
                node = node.Children[c];
            
            // Проверка всех слов, заканчивающихся в текущей вершине и выше по суффиксным ссылкам
            var temp = node;
            while (temp != _root)
            {
                if (temp.IsEnd)
                {
                    foreach (var word in temp.Words)
                    {
                        // Проверка: начинается и заканчивается на гласную
                        if (_vowels.Contains(temp.FirstChar) && _vowels.Contains(c))
                        {
                            result.Add((i - word.Length + 1, word));
                        }
                    }
                }
                temp = temp.SuffixLink!;
            }
        }
        
        return result;
    }
}
```

**Ответ на вопрос:**

Можно ли адаптировать автомат для немедленной фильтрации? **Да**, и это делается двумя способами:

1. **Проверка при обходе** (показано выше): при нахождении вхождения проверяем первый и последний символы. Это не требует изменения структуры автомата, только дополнительной проверки при выдаче результата.

2. **Предварительная фильтрация шаблонов:** ещё до построения автомата отбросить все ключевые слова, которые не начинаются или не заканчиваются на гласную. Тогда автомат будет строить только для подходящих слов, и проверка при обходе станет проще (нужно проверять только последний символ, так как первый уже гарантированно гласный).

**Оценка сложности:**
- Построение бора: $O(\sum |p_i|)$, где $|p_i|$ — длины ключевых слов.
- Построение суффиксных ссылок: $O(\sum |p_i| \cdot |\Sigma|)$ или $O(\sum |p_i|)$ при использовании хеш-таблицы для детей.
- Обработка текста: $O(|t| + k)$, где $k$ — число найденных вхождений.

### 6. Суффиксный массив и количество различных подстрок

**Условие:** Дана строка $s$ длины $n$. Требуется вычислить количество различных подстрок строки $s$. Опишите, как использовать суффиксный массив и LCP-массив для решения этой задачи.

**Решение:**

**Суффиксный массив** — это массив перестановки индексов всех суффиксов строки, отсортированных в лексикографическом порядке. Если $SA[i] = j$, то $i$-й суффикс в отсортированном порядке начинается с позиции $j$ в исходной строке.

**LCP-массив** (Longest Common Prefix) хранит длины наибольших общих префиксов между соседними суффиксами в отсортированном порядке: $LCP[i]$ — длина наибольшего общего префикса суффиксов $SA[i-1]$ и $SA[i]$.

**Почему формула работает:**

Общее количество подстрок строки длины $n$ равно $\frac{n(n+1)}{2}$ (каждая подстрока определяется началом и концом). Однако многие подстроки могут повторяться.

Ключевое наблюдение: каждая подстрока строки $s$ является префиксом некоторого суффикса. Если мы рассмотрим все суффиксы в отсортированном порядке, то:
- Суффикс $SA[i]$ имеет длину $n - SA[i]$ и, соответственно, столько же префиксов (подстрок, начинающихся в этой позиции).
- Но $LCP[i]$ префиксов этого суффикса уже встречались как префиксы предыдущего суффикса $SA[i-1]$ (так как у них общий префикс длины $LCP[i]$).
- Значит, суффикс $SA[i]$ добавляет ровно $(n - SA[i]) - LCP[i]$ **новых** различных подстрок.

Суммируя по всем суффиксам:

$$\text{count} = \sum_{i=0}^{n-1} (n - SA[i]) - \sum_{i=1}^{n-1} LCP[i]$$

Первое слагаемое — это просто сумма длин всех суффиксов, которая равна $\frac{n(n+1)}{2}$. Таким образом:

$$\text{count} = \frac{n(n+1)}{2} - \sum_{i=1}^{n-1} LCP[i]$$

**Пример для строки "ababa":**

$n = 5$, общее количество подстрок без учёта повторений: $\frac{5 \cdot 6}{2} = 15$.

Суффиксы:
- 0: "ababa"
- 1: "baba"
- 2: "aba"
- 3: "ba"
- 4: "a"

Отсортированные суффиксы (суффиксный массив):
- $SA[0] = 4$: "a"
- $SA[1] = 2$: "aba"
- $SA[2] = 0$: "ababa"
- $SA[3] = 3$: "ba"
- $SA[4] = 1$: "baba"

LCP-массив:
- $LCP[1] = \text{lcp}("a", "aba") = 1$
- $LCP[2] = \text{lcp}("aba", "ababa") = 3$
- $LCP[3] = \text{lcp}("ababa", "ba") = 0$
- $LCP[4] = \text{lcp}("ba", "baba") = 2$

$\sum LCP = 1 + 3 + 0 + 2 = 6$

Количество различных подстрок: $15 - 6 = 9$.

Проверим вручную: "a", "ab", "aba", "abab", "ababa", "b", "ba", "bab", "baba" — действительно 9.

**Псевдокод:**

```csharp
public static int CountDistinctSubstrings(string s)
{
    int n = s.Length;
    if (n == 0) return 0;
    
    // Построение суффиксного массива (O(n log^2 n) или O(n log n))
    int[] sa = BuildSuffixArray(s);
    
    // Построение LCP-массива (O(n))
    int[] lcp = BuildLcpArray(s, sa);
    
    // Формула
    int totalSubstrings = n * (n + 1) / 2;
    int lcpSum = 0;
    for (int i = 1; i < n; i++)
        lcpSum += lcp[i];
    
    return totalSubstrings - lcpSum;
}

// Упрощённая реализация суффиксного массива (O(n log^2 n))
private static int[] BuildSuffixArray(string s)
{
    int n = s.Length;
    var suffixes = new (string suffix, int index)[n];
    
    for (int i = 0; i < n; i++)
        suffixes[i] = (s.Substring(i), i);
    
    Array.Sort(suffixes, (a, b) => string.Compare(a.suffix, b.suffix, StringComparison.Ordinal));
    
    var sa = new int[n];
    for (int i = 0; i < n; i++)
        sa[i] = suffixes[i].index;
    
    return sa;
}

// Построение LCP-массива за O(n) с использованием обратной перестановки
private static int[] BuildLcpArray(string s, int[] sa)
{
    int n = s.Length;
    var rank = new int[n]; // Обратная перестановка: rank[sa[i]] = i
    
    for (int i = 0; i < n; i++)
        rank[sa[i]] = i;
    
    var lcp = new int[n];
    int k = 0;
    
    for (int i = 0; i < n; i++)
    {
        if (rank[i] == 0)
        {
            k = 0;
            continue;
        }
        
        int j = sa[rank[i] - 1]; // Предыдущий суффикс в отсортированном порядке
        
        while (i + k < n && j + k < n && s[i + k] == s[j + k])
            k++;
        
        lcp[rank[i]] = k;
        
        if (k > 0)
            k--;
    }
    
    return lcp;
}
```

**Оценка сложности:**
- Построение суффиксного массива: $O(n \log^2 n)$ наивной сортировкой или $O(n \log n)$ с удвоением префиксов.
- Построение LCP-массива: $O(n)$ с использованием обратной перестановки.
- Подсчёт суммы: $O(n)$.
- **Итого:** $O(n \log n)$ или $O(n \log^2 n)$ в зависимости от реализации.

### 7. Алгоритм Куна (паросочетания в двудольном графе)

**Условие:** В некоторой школе есть $n$ учеников и $m$ кружков. Каждый ученик может посещать несколько кружков, но в итоге каждый кружок может принять только одного ученика, и каждый ученик может ходить только в один кружок. Напишите псевдокод алгоритма Куна, который находит максимальное количество назначений.

**Решение:**

**Паросочетание** в графе — это набор рёбер, не имеющих общих вершин. **Максимальное паросочетание** — паросочетание наибольшего возможного размера. В двудольном графе (вершины разделены на две доли, и рёбра соединяют только вершины из разных долей) задача нахождения максимального паросочетания решается алгоритмом Куна.

**Основная идея алгоритма Куна:**

Алгоритм пытается последовательно для каждой вершины левой доли найти **дополняющий путь** — путь, который начинается в свободной вершине левой доли, заканчивается в свободной вершине правой доли, и рёбра на котором чередуются: не из паросочетания, из паросочетания, не из паросочетания, и т.д. Если такой путь найден, можно «перевернуть» рёбра на пути (рёбра не из паросочетания добавить, рёбра из паросочетания удалить), и размер паросочетания увеличится на 1.

**Алгоритм:**

1. Инициализируем паросочетание пустым.
2. Для каждой вершины $v$ в левой доле:
   - Пытаемся найти дополняющий путь из $v$ с помощью поиска в глубину (DFS).
   - Если путь найден, увеличиваем паросочетание.
3. Возвращаем размер паросочетания.

**Псевдокод:**

```csharp
public class KuhnAlgorithm
{
    private readonly List<List<int>> _graph; // Список смежности для левой доли
    private readonly int _n; // Количество вершин в левой доле
    private readonly int _m; // Количество вершин в правой доле
    
    public KuhnAlgorithm(int n, int m)
    {
        _n = n;
        _m = m;
        _graph = new List<List<int>>();
        for (int i = 0; i < n; i++)
            _graph.Add(new List<int>());
    }
    
    public void AddEdge(int student, int club)
    {
        _graph[student].Add(club);
    }
    
    public (int matchingSize, int[] matchR) FindMaxMatching()
    {
        // matchR[u] = v означает, что вершина u правой доли спарена с v из левой
        var matchR = new int[_m];
        Array.Fill(matchR, -1);
        
        int result = 0;
        
        for (int v = 0; v < _n; v++)
        {
            // used — массив посещённых вершин левой доли в текущем DFS
            var used = new bool[_n];
            if (Dfs(v, used, matchR))
                result++;
        }
        
        return (result, matchR);
    }
    
    private bool Dfs(int v, bool[] used, int[] matchR)
    {
        if (used[v])
            return false;
        
        used[v] = true;
        
        foreach (int u in _graph[v])
        {
            // Если u свободна или удалось найти альтернативный путь для matchR[u]
            if (matchR[u] == -1 || Dfs(matchR[u], used, matchR))
            {
                matchR[u] = v;
                return true;
            }
        }
        
        return false;
    }
}
```

**Пример использования:**

```csharp
// 3 ученика, 3 кружка
var kuhn = new KuhnAlgorithm(3, 3);
kuhn.AddEdge(0, 0); // Ученик 0 может в кружок 0
kuhn.AddEdge(0, 1); // Ученик 0 может в кружок 1
kuhn.AddEdge(1, 0); // Ученик 1 может в кружок 0
kuhn.AddEdge(1, 2); // Ученик 1 может в кружок 2
kuhn.AddEdge(2, 1); // Ученик 2 может в кружок 1

var (size, matchR) = kuhn.FindMaxMatching();
Console.WriteLine($"Максимальное паросочетание: {size}");
// matchR[i] = номер ученика в кружке i, или -1 если кружок свободен
```

**Оценка сложности:**

- Каждый вызов DFS просматривает $O(E)$ рёбер в худшем случае.
- DFS вызывается $O(V)$ раз (по одному на каждую вершину левой доли).
- **Итого:** $O(V \cdot E)$, где $V = n + m$ — общее число вершин, $E$ — число рёбер.

**Можно ли ускорить для разреженных графов?**

Да, для разреженных графов (когда $E \ll V^2$) алгоритм Куна часто работает быстрее теоретической оценки. Дополнительные оптимизации:

1. **Жадная инициализация:** перед запуском основного алгоритма можно жадно добавить все возможные рёбра в паросочетание. Это уменьшит число итераций DFS.

2. **Оптимизация с персистентным массивом посещений:** вместо создания нового массива `used` на каждой итерации можно использовать один массив с «меткой итерации».

3. **Алгоритм Хопкрофта-Карпа:** более сложный алгоритм со сложностью $O(E \sqrt{V})$, который находит несколько дополняющих путей за одну итерацию с помощью BFS.

```csharp
// Оптимизация: жадная инициализация
public (int, int[]) FindMaxMatchingWithGreedyInit()
{
    var matchR = new int[_m];
    Array.Fill(matchR, -1);
    var matchL = new int[_n];
    Array.Fill(matchL, -1);
    
    // Жадное начальное паросочетание
    for (int v = 0; v < _n; v++)
    {
        foreach (int u in _graph[v])
        {
            if (matchR[u] == -1)
            {
                matchR[u] = v;
                matchL[v] = u;
                break;
            }
        }
    }
    
    int result = matchR.Count(x => x != -1);
    
    // Основной алгоритм только для свободных вершин
    for (int v = 0; v < _n; v++)
    {
        if (matchL[v] == -1)
        {
            var used = new bool[_n];
            if (Dfs(v, used, matchR))
                result++;
        }
    }
    
    return (result, matchR);
}
```

### 8. Алгоритм Эдмондса для общих графов (цветки)

**Условие:** В компании планируется организовать парные встречи сотрудников. Отношения между сотрудниками заданы неориентированным графом. Нужно найти максимальное количество пар. Объясните, почему в недвудольном графе нельзя применить алгоритм Куна. В чём состоит понятие цветка?

**Решение:**

**Почему алгоритм Куна не работает для общих графов:**

Алгоритм Куна опирается на свойство двудольных графов: любой цикл в двудольном графе имеет чётную длину. Это гарантирует, что при поиске дополняющего пути мы не столкнёмся с ситуацией, когда в вершину можно прийти двумя путями разной чётности.

В общем (недвудольном) графе могут существовать циклы нечётной длины. Это создаёт проблему: при поиске дополняющего пути DFS может «застрять», пройдя по циклу нечётной длины и вернувшись в ту же вершину с другой чётностью пути. Простой пример — граф-треугольник с висячим ребром:

```
    1
   / \
  0---2
  |
  3
```

Здесь есть дополняющий путь 3-0-1-2, но DFS из вершины 3 может не найти его, если сначала пойдёт по циклу 0-1-2-0.

**Понятие цветка:**

**Цветок (blossom)** — это цикл нечётной длины в графе, к которому присоединён «стебель» — путь чётной длины от свободной вершины до одной из вершин цикла. Ключевое свойство: если в графе есть цветок, то можно найти дополняющий путь тогда и только тогда, когда он существует в графе после **сжатия цветка** в одну вершину.

**Алгоритм Эдмондса:**

1. Ищем дополняющий путь с помощью поиска в ширину (BFS), строя чередующееся дерево.
2. Если при обходе обнаруживаем ребро между двумя вершинами одного уровня дерева (это указывает на цикл нечётной длины), то нашли цветок.
3. **Сжимаем цветок:** заменяем все вершины цветка на одну новую вершину, переподключая все внешние рёбра к новой вершине.
4. Продолжаем поиск в сжатом графе.
5. Если дополняющий путь найден, **раскручиваем** сжатия обратно, восстанавливая путь в исходном графе.

**Псевдокод (упрощённо):**

```csharp
public class EdmondsAlgorithm
{
    private readonly int _n;
    private readonly List<List<int>> _graph;
    
    public EdmondsAlgorithm(int n)
    {
        _n = n;
        _graph = new List<List<int>>();
        for (int i = 0; i < n; i++)
            _graph.Add(new List<int>());
    }
    
    public void AddEdge(int u, int v)
    {
        _graph[u].Add(v);
        _graph[v].Add(u);
    }
    
    public List<(int, int)> FindMaxMatching()
    {
        var match = new int[_n];
        Array.Fill(match, -1);
        
        for (int start = 0; start < _n; start++)
        {
            if (match[start] != -1) continue;
            
            // BFS для поиска дополняющего пути
            var parent = new int[_n];
            Array.Fill(parent, -1);
            
            var queue = new Queue<int>();
            queue.Enqueue(start);
            
            var used = new bool[_n];
            used[start] = true;
            
            int endPoint = -1;
            
            while (queue.Count > 0 && endPoint == -1)
            {
                int u = queue.Dequeue();
                
                foreach (int v in _graph[u])
                {
                    if (!used[v])
                    {
                        used[v] = true;
                        parent[v] = u;
                        
                        if (match[v] == -1)
                        {
                            endPoint = v;
                            break;
                        }
                        
                        // Переходим к спаренной вершине
                        int w = match[v];
                        if (!used[w])
                        {
                            used[w] = true;
                            parent[w] = v;
                            queue.Enqueue(w);
                        }
                    }
                    else if (parent[u] != v && parent[v] == -1)
                    {
                        // Обнаружен цветок — требуется сжатие
                        // (реализация сжатия опущена для краткости)
                        // В полной реализации здесь вызывается процедура сжатия
                    }
                }
            }
            
            if (endPoint != -1)
            {
                // Инвертируем рёбра на дополняющем пути
                int cur = endPoint;
                while (cur != -1 && parent[cur] != -1)
                {
                    int prev = parent[cur];
                    int next = match[prev];
                    match[cur] = prev;
                    match[prev] = cur;
                    cur = next;
                }
            }
        }
        
        var result = new List<(int, int)>();
        for (int i = 0; i < _n; i++)
        {
            if (match[i] != -1 && i < match[i])
                result.Add((i, match[i]));
        }
        
        return result;
    }
}
```

**Пример графа с цветком:**

Рассмотрим граф из 5 вершин:
- Рёбра: (0,1), (1,2), (2,0) — треугольник (цветок), (0,3), (3,4) — стебель.

```
    1
   / \
  0---2
  |
  3
  |
  4 (свободна)
```

Предположим, что вершина 4 свободна. При поиске дополняющего пути из 4:
1. Идём 4 → 3 → 0.
2. Из 0 можно пойти в 1 или 2.
3. Если пойдём 0 → 1 → 2 → 0, то вернёмся в 0 с другой чётностью — это цветок.

**Сжатие цветка:** заменяем вершины {0, 1, 2} на одну вершину B. Теперь граф:
- B соединена с 3.
- 3 соединена с 4.

Поиск продолжается в этом сжатом графе.

**Оценка сложности:** $O(V^3)$ наивной реализацией или $O(V \cdot E)$ с эффективными структурами данных.

### 9. Форд-Фалкерсон и теорема о максимальном потоке и минимальном разрезе

**Условие:** Дана сеть с истоком $s$ и стоком $t$. После выполнения алгоритма Форда-Фалкерсона получен максимальный поток. Как по остаточной сети найти минимальный разрез?

**Решение:**

**Основные понятия:**

- **Сеть** — ориентированный граф $G = (V, E)$ с выделенными истоком $s$ и стоком $t$, где каждое ребро $(u, v)$ имеет пропускную способность $c(u, v) \ge 0$.
- **Поток** — функция $f: V \times V \to \mathbb{R}$, удовлетворяющая ограничениям пропускной способности ($0 \le f(u, v) \le c(u, v)$) и закону сохранения (сумма входящих потоков равна сумме выходящих для всех вершин, кроме $s$ и $t$).
- **Остаточная сеть** $G_f$ — граф с теми же вершинами, где каждое ребро $(u, v)$ имеет остаточную пропускную способность $c_f(u, v) = c(u, v) - f(u, v)$. Если $f(u, v) > 0$, появляется обратное ребро $(v, u)$ с пропускной способностью $f(u, v)$.
- **Разрез** — разбиение вершин на два множества $(S, T)$, где $s \in S$, $t \in T$. **Пропускная способность разреза** — сумма пропускных способностей всех рёбер из $S$ в $T$.

**Теорема о максимальном потоке и минимальном разрезе:**

> Величина максимального потока из $s$ в $t$ равна пропускной способности минимального разреза, разделяющего $s$ и $t$.

**Как найти минимальный разрез после Форда-Фалкерсона:**

1. После завершения алгоритма Форда-Фалкерсона (когда в остаточной сети нет пути из $s$ в $t$) запускаем BFS или DFS из $s$ в остаточной сети.
2. Все вершины, достижимые из $s$, образуют множество $S$.
3. Все остальные вершины образуют множество $T = V \setminus S$.
4. Рёбра, идущие из $S$ в $T$ в исходной сети, образуют минимальный разрез.

**Почему это работает:**

После завершения Форда-Фалкерсона в остаточной сети нет пути из $s$ в $t$ (иначе алгоритм продолжил бы работу). Значит, множество достижимых из $s$ вершин $S$ не содержит $t$, и $(S, V \setminus S)$ — корректный разрез.

Все рёбра из $S$ в $T$ в исходной сети **насыщены** (их остаточная пропускная способность равна 0), иначе они были бы в остаточной сети и вершины из $T$ были бы достижимы. Все рёбра из $T$ в $S$ имеют нулевой поток (иначе в остаточной сети были бы обратные рёбра).

Таким образом, поток через разрез равен сумме пропускных способностей рёбер из $S$ в $T$, что по закону сохранения равно величине потока из $s$ в $t$.

**Псевдокод:**

```csharp
public class FordFulkerson
{
    private readonly int _n;
    private readonly int[,] _capacity; // Пропускные способности
    private readonly List<List<int>> _adj; // Список смежности
    
    public FordFulkerson(int n)
    {
        _n = n;
        _capacity = new int[n, n];
        _adj = new List<List<int>>();
        for (int i = 0; i < n; i++)
            _adj.Add(new List<int>());
    }
    
    public void AddEdge(int u, int v, int cap)
    {
        _adj[u].Add(v);
        _adj[v].Add(u); // Обратное ребро для остаточной сети
        _capacity[u, v] = cap;
    }
    
    public (int maxFlow, (List<int> S, List<int> T) minCut) ComputeMaxFlowAndMinCut(int s, int t)
    {
        var flow = new int[_n, _n];
        int maxFlow = 0;
        
        // Алгоритм Форда-Фалкерсона с BFS (Эдмондса-Карпа)
        while (true)
        {
            var parent = new int[_n];
            Array.Fill(parent, -1);
            
            var queue = new Queue<int>();
            queue.Enqueue(s);
            
            while (queue.Count > 0 && parent[t] == -1)
            {
                int u = queue.Dequeue();
                foreach (int v in _adj[u])
                {
                    if (parent[v] == -1 && _capacity[u, v] - flow[u, v] > 0)
                    {
                        parent[v] = u;
                        queue.Enqueue(v);
                    }
                }
            }
            
            if (parent[t] == -1)
                break; // Пути нет
            
            // Находим минимальную пропускную способность на пути
            int push = int.MaxValue;
            int cur = t;
            while (cur != s)
            {
                int prev = parent[cur];
                push = Math.Min(push, _capacity[prev, cur] - flow[prev, cur]);
                cur = prev;
            }
            
            // Обновляем поток
            cur = t;
            while (cur != s)
            {
                int prev = parent[cur];
                flow[prev, cur] += push;
                flow[cur, prev] -= push;
                cur = prev;
            }
            
            maxFlow += push;
        }
        
        // Находим минимальный разрез: BFS из s в остаточной сети
        var visited = new bool[_n];
        var queue2 = new Queue<int>();
        queue2.Enqueue(s);
        visited[s] = true;
        
        while (queue2.Count > 0)
        {
            int u = queue2.Dequeue();
            foreach (int v in _adj[u])
            {
                if (!visited[v] && _capacity[u, v] - flow[u, v] > 0)
                {
                    visited[v] = true;
                    queue2.Enqueue(v);
                }
            }
        }
        
        var S = new List<int>();
        var T = new List<int>();
        for (int i = 0; i < _n; i++)
        {
            if (visited[i])
                S.Add(i);
            else
                T.Add(i);
        }
        
        return (maxFlow, (S, T));
    }
}
```

**Пример:**

Рассмотрим сеть:
- $s = 0$, $t = 3$.
- Рёбра: $(0, 1)$ с $c=10$, $(0, 2)$ с $c=5$, $(1, 2)$ с $c=15$, $(1, 3)$ с $c=10$, $(2, 3)$ с $c=10$.

После Форда-Фалкерсона максимальный поток будет 15.

Остаточная сеть:
- Из 0 достижимы: 0, 1 (через остаточное ребро), 2 (через остаточное ребро).
- 3 недостижима.

Минимальный разрез: $S = \{0, 1, 2\}$, $T = \{3\}$.
Пропускная способность: $c(1, 3) + c(2, 3) = 10 + 10 = 20$... стоп, это не сходится с потоком 15.

На самом деле, после насыщения рёбер достижимые вершины могут быть другими. В данном примере правильный разрез: $S = \{0\}$, $T = \{1, 2, 3\}$, пропускная способность $c(0, 1) + c(0, 2) = 10 + 5 = 15$, что равно потоку.

### 10. Алгоритм Диница (слоистая сеть и блокирующий поток)

**Условие:** Почему алгоритм Эдмондса-Карпа может быть неэффективным для сетей с большими пропускными способностями? Объясните идею алгоритма Диница.

**Решение:**

**Проблема Эдмондса-Карпа:**

Алгоритм Эдмондса-Карпа на каждом шаге находит **один** кратчайший по числу рёбер дополняющий путь с помощью BFS и увеличивает поток вдоль него. Проблема в том, что после этого вся информация, полученная при BFS (все другие кратчайшие пути), отбрасывается. Если в сети много кратчайших путей, алгоритм будет многократно выполнять BFS, находя по одному пути за раз.

Для сетей с большими пропускными способностями это особенно плохо: даже если кратчайший путь найден быстро, увеличение потока может быть небольшим (ограничено «узким местом» на пути), и потребуется много итераций.

**Идея алгоритма Диница:**

Вместо поиска одного пути за раз, алгоритм Диница находит **все** кратчайшие пути одновременно и насыщает их за одну фазу. Это достигается с помощью понятия **слоистой сети** и **блокирующего потока**.

**Слоистая сеть (Level Graph):**

1. Запускаем BFS из истока $s$ в остаточной сети и вычисляем расстояние $dist[v]$ — минимальное число рёбер от $s$ до $v$.
2. Строим новый граф $G_L$, включающий только те рёбра $(u, v)$ из остаточной сети, для которых $dist[v] = dist[u] + 1$. Такие рёбра лежат на каких-то кратчайших путях.

В слоистой сети любой путь из $s$ в $t$ является кратчайшим в исходной сети.

**Блокирующий поток:**

Блокирующий поток в слоистой сети — это поток, который насыщает **все** пути из $s$ в $t$ в этой сети. То есть после пуска блокирующего потока в слоистой сети не остаётся ни одного пути из $s$ в $t$.

**Алгоритм Диница:**

1. Инициализируем поток нулём.
2. **Фаза:**
   - Строим слоистую сеть с помощью BFS.
   - Если $t$ недостижима, завершаем (поток максимален).
   - Находим блокирующий поток в слоистой сети с помощью DFS.
   - Добавляем блокирующий поток к общему потоку.
3. Повторяем фазу.

**Поиск блокирующего потока с DFS и удалением тупиков:**

Вместо явного построения слоистой сети и удаления тупиков, можно использовать оптимизированный DFS:
- Запускаем DFS из $s$.
- Если из вершины $u$ не удалось достичь $t$, помечаем ребро, по которому вошли в $u$, как «бесполезное» и больше по нему не идём.
- Это эквивалентно удалению тупиков, но без явного изменения графа.

**Псевдокод:**

```csharp
public class DinicAlgorithm
{
    private readonly int _n;
    private readonly List<List<Edge>> _graph;
    
    private class Edge
    {
        public int To;
        public int Capacity;
        public int Flow;
        public int ReverseEdgeIndex; // Индекс обратного ребра в _graph[To]
        
        public Edge(int to, int capacity, int revIdx)
        {
            To = to;
            Capacity = capacity;
            Flow = 0;
            ReverseEdgeIndex = revIdx;
        }
        
        public int RemainingCapacity => Capacity - Flow;
    }
    
    public DinicAlgorithm(int n)
    {
        _n = n;
        _graph = new List<List<Edge>>();
        for (int i = 0; i < n; i++)
            _graph.Add(new List<Edge>());
    }
    
    public void AddEdge(int from, int to, int capacity)
    {
        var forward = new Edge(to, capacity, _graph[to].Count);
        var backward = new Edge(from, 0, _graph[from].Count); // Обратное ребро с capacity 0
        
        _graph[from].Add(forward);
        _graph[to].Add(backward);
    }
    
    public int ComputeMaxFlow(int s, int t)
    {
        int maxFlow = 0;
        var level = new int[_n];
        
        while (true)
        {
            // Построение слоистой сети (BFS)
            Array.Fill(level, -1);
            level[s] = 0;
            
            var queue = new Queue<int>();
            queue.Enqueue(s);
            
            while (queue.Count > 0)
            {
                int u = queue.Dequeue();
                foreach (var edge in _graph[u])
                {
                    if (edge.RemainingCapacity > 0 && level[edge.To] == -1)
                    {
                        level[edge.To] = level[u] + 1;
                        queue.Enqueue(edge.To);
                    }
                }
            }
            
            // Если сток недостижим, завершаем
            if (level[t] == -1)
                break;
            
            // Поиск блокирующего потока (DFS с оптимизацией)
            var ptr = new int[_n]; // Указатель на следующее ребро для проверки
            while (true)
            {
                int pushed = Dfs(s, t, int.MaxValue, level, ptr);
                if (pushed == 0)
                    break;
                maxFlow += pushed;
            }
        }
        
        return maxFlow;
    }
    
    private int Dfs(int u, int t, int flow, int[] level, int[] ptr)
    {
        if (flow == 0 || u == t)
            return flow;
        
        for (int i = ptr[u]; i < _graph[u].Count; i++)
        {
            ptr[u] = i; // Обновляем указатель, чтобы не проверять пройденные рёбра
            var edge = _graph[u][i];
            
            if (level[edge.To] != level[u] + 1 || edge.RemainingCapacity == 0)
                continue;
            
            int pushed = Dfs(edge.To, t, Math.Min(flow, edge.RemainingCapacity), level, ptr);
            
            if (pushed > 0)
            {
                edge.Flow += pushed;
                _graph[edge.To][edge.ReverseEdgeIndex].Flow -= pushed;
                return pushed;
            }
        }
        
        return 0;
    }
}
```

**Оценка сложности:**

- Построение слоистой сети: $O(E)$.
- Поиск блокирующего потока: $O(V \cdot E)$ наивным DFS, но с оптимизацией (указатели `ptr`) — $O(E)$ на фазу.
- Число фаз: $O(V)$, так как расстояние до $t$ строго увеличивается после каждой фазы.
- **Итого:** $O(V \cdot E^2)$ наивной реализацией или $O(V^2 \cdot E)$ с оптимизацией.

Для сетей с единичными пропускными способностями сложность ещё лучше: $O(\min(V^{2/3}, E^{1/2}) \cdot E)$.

### 11. Венгерский алгоритм (задача о назначениях)

**Условие:** Дана матрица стоимости $C$ размера $n \times n$. Необходимо назначить каждому работнику ровно одну работу так, чтобы суммарная стоимость была минимальна.

**Решение:**

**Задача о назначениях** — это частный случай транспортной задачи, где $n$ работников должны быть назначены на $n$ работ, и каждый работник может выполнить любую работу с некоторой стоимостью $C[i][j]$. Требуется найти назначение с минимальной суммарной стоимостью.

**Основная идея венгерского алгоритма:**

Алгоритм основан на следующем наблюдении: если из всех элементов строки (или столбца) матрицы вычесть одно и то же число, оптимальное назначение не изменится (изменится только суммарная стоимость на константу). Это позволяет приводить матрицу к виду с нулями, и затем искать полное паросочетание по нулевым элементам.

**Шаги алгоритма:**

1. **Приведение матрицы:**
   - Из каждой строки вычитаем её минимальный элемент.
   - Из каждого столбца вычитаем его минимальный элемент.
   - Теперь в каждой строке и столбце есть хотя бы один ноль.

2. **Поиск максимального паросочетания по нулям:**
   - Строим двудольный граф, где ребро $(i, j)$ существует, если $C'[i][j] = 0$.
   - Находим максимальное паросочетание алгоритмом Куна.
   - Если паросочетание полное (размера $n$), оно оптимально — завершаем.

3. **Модификация матрицы (если паросочетание неполное):**
   - Находим минимальное число линий (строк и столбцов), покрывающих все нули.
   - Находим минимальный элемент среди непокрытых.
   - Вычитаем его из всех непокрытых элементов и добавляем к элементам на пересечении линий.
   - Возвращаемся к шагу 2.

**Пример для матрицы:**

$$C = \begin{pmatrix} 5 & 8 & 7 \\ 6 & 4 & 9 \\ 3 & 2 & 6 \end{pmatrix}$$

**Шаг 1: Приведение строк**

Минимумы строк: 5, 4, 3.

$$\begin{pmatrix} 0 & 3 & 2 \\ 2 & 0 & 5 \\ 0 & 0 & 3 \end{pmatrix}$$

**Шаг 2: Приведение столбцов**

Минимумы столбцов: 0, 0, 2.

$$\begin{pmatrix} 0 & 3 & 0 \\ 2 & 0 & 3 \\ 0 & 0 & 1 \end{pmatrix}$$

**Шаг 3: Поиск паросочетания по нулям**

Нули на позициях: (0,0), (0,2), (1,1), (2,0), (2,1).

Пытаемся найти полное паросочетание:
- Работник 0 → работа 0
- Работник 1 → работа 1
- Работник 2 → работа 2? Но (2,2) не ноль.

Пробуем по-другому:
- 0 → 2
- 1 → 1
- 2 → 0

Это полное паросочетание по нулям! Стоимость в приведённой матрице: $0 + 0 + 0 = 0$.

**Шаг 4: Восстановление стоимости**

Исходная стоимость: $C[0][2] + C[1][1] + C[2][0] = 7 + 4 + 3 = 14$.

**Псевдокод:**

```csharp
public class HungarianAlgorithm
{
    public static (int cost, int[] assignment) Solve(int[,] costMatrix)
    {
        int n = costMatrix.GetLength(0);
        var u = new int[n + 1]; // Потенциалы строк
        var v = new int[n + 1]; // Потенциалы столбцов
        var p = new int[n + 1]; // p[j] = строка, назначенная на столбец j
        var way = new int[n + 1]; // Для восстановления пути
        
        for (int i = 1; i <= n; i++)
        {
            p[0] = i;
            var minv = new int[n + 1];
            Array.Fill(minv, int.MaxValue);
            var used = new bool[n + 1];
            int j0 = 0;
            
            do
            {
                used[j0] = true;
                int i0 = p[j0], delta = int.MaxValue, j1 = 0;
                
                for (int j = 1; j <= n; j++)
                {
                    if (!used[j])
                    {
                        int cur = costMatrix[i0 - 1, j - 1] - u[i0] - v[j];
                        if (cur < minv[j])
                        {
                            minv[j] = cur;
                            way[j] = j0;
                        }
                        if (minv[j] < delta)
                        {
                            delta = minv[j];
                            j1 = j;
                        }
                    }
                }
                
                for (int j = 0; j <= n; j++)
                {
                    if (used[j])
                    {
                        u[p[j]] += delta;
                        v[j] -= delta;
                    }
                    else
                    {
                        minv[j] -= delta;
                    }
                }
                
                j0 = j1;
            } while (p[j0] != 0);
            
            do
            {
                int j1 = way[j0];
                p[j0] = p[j1];
                j0 = j1;
            } while (j0 != 0);
        }
        
        var assignment = new int[n];
        for (int j = 1; j <= n; j++)
        {
            if (p[j] != 0)
                assignment[p[j] - 1] = j - 1;
        }
        
        int totalCost = 0;
        for (int i = 0; i < n; i++)
            totalCost += costMatrix[i, assignment[i]];
        
        return (totalCost, assignment);
    }
}

// Пример использования:
var matrix = new int[,] {
    {5, 8, 7},
    {6, 4, 9},
    {3, 2, 6}
};

var (cost, assignment) = HungarianAlgorithm.Solve(matrix);
Console.WriteLine($"Минимальная стоимость: {cost}");
Console.WriteLine($"Назначение: [{string.Join(", ", assignment)}]");
// Вывод: стоимость 14, назначение [2, 1, 0]
```

**Оценка сложности:** $O(n^3)$.

### 12. Поток минимальной стоимости (метод потенциалов)

**Условие:** В сети каждое ребро имеет пропускную способность и стоимость. Требуется отправить из $s$ в $t$ заданный объём потока $F$ с минимальной суммарной стоимостью.

**Решение:**

**Постановка задачи:**

В отличие от задачи о максимальном потоке, здесь каждое ребро $(u, v)$ имеет не только пропускную способность $c(u, v)$, но и стоимость $w(u, v)$ за единицу потока. Требуется найти поток величины $F$ с минимальной суммарной стоимостью $\sum f(u, v) \cdot w(u, v)$.

**Проблема отрицательных рёбер:**

При увеличении потока вдоль пути в остаточной сети появляются обратные рёбра с **отрицательной** стоимостью. Действительно, если мы пускаем поток по ребру $(u, v)$ со стоимостью $w$, то обратное ребро $(v, u)$ имеет стоимость $-w$ (уменьшение потока равносильно «возврату» стоимости).

Это создаёт проблему: алгоритм Дейкстры для поиска кратчайшего пути не работает с отрицательными весами. Пришлось бы использовать Форда-Беллмана за $O(V \cdot E)$ на каждой итерации, что медленно.

**Метод потенциалов:**

Идея: пересчитать стоимости рёбер так, чтобы они стали неотрицательными, но кратчайшие пути сохранились.

Для каждой вершины $v$ введём **потенциал** $h(v)$. **Пересчитанная стоимость** ребра $(u, v)$:

$$w'(u, v) = w(u, v) + h(u) - h(v)$$

**Ключевые свойства:**

1. Если все $w' \ge 0$, можно использовать Дейкстру.
2. Кратчайшие пути относительно $w'$ совпадают с кратчайшими относительно $w$ (так как разность потенциалов на пути сокращается).
3. После увеличения потока обновляем потенциалы: $h(v) := h(v) + dist(v)$, где $dist(v)$ — расстояние от $s$ до $v$ в пересчитанных весах.

**Алгоритм:**

1. Инициализируем поток нулём, потенциалы — нулями (или вычисляем Фордом-Беллманом, если есть отрицательные рёбра в исходной сети).
2. Пока поток меньше $F$:
   - Пересчитываем стоимости: $w'(u, v) = w(u, v) + h(u) - h(v)$.
   - Находим кратчайший путь из $s$ в $t$ алгоритмом Дейкстры с весами $w'$.
   - Если пути нет, завершаем (нельзя достичь $F$).
   - Увеличиваем поток вдоль пути на максимально возможную величину.
   - Обновляем потенциалы: $h(v) := h(v) + dist(v)$.
3. Возвращаем поток и стоимость.

**Псевдокод:**

```csharp
public class MinCostMaxFlow
{
    private readonly int _n;
    private readonly List<List<Edge>> _graph;
    
    private class Edge
    {
        public int To;
        public int Capacity;
        public int Flow;
        public int Cost;
        public int ReverseEdgeIndex;
        
        public Edge(int to, int capacity, int cost, int revIdx)
        {
            To = to;
            Capacity = capacity;
            Cost = cost;
            ReverseEdgeIndex = revIdx;
            Flow = 0;
        }
    }
    
    public MinCostMaxFlow(int n)
    {
        _n = n;
        _graph = new List<List<Edge>>();
        for (int i = 0; i < n; i++)
            _graph.Add(new List<Edge>());
    }
    
    public void AddEdge(int from, int to, int capacity, int cost)
    {
        var forward = new Edge(to, capacity, cost, _graph[to].Count);
        var backward = new Edge(from, 0, -cost, _graph[from].Count);
        
        _graph[from].Add(forward);
        _graph[to].Add(backward);
    }
    
    public (int flow, int cost) ComputeMinCostFlow(int s, int t, int requiredFlow)
    {
        int flow = 0, totalCost = 0;
        var h = new int[_n]; // Потенциалы
        
        // Если есть отрицательные рёбра, инициализируем потенциалы Фордом-Беллманом
        // (опущено для краткости, предполагаем неотрицательные исходные стоимости)
        
        while (flow < requiredFlow)
        {
            // Дейкстра с пересчитанными весами
            var dist = new int[_n];
            var parent = new int[_n];
            var parentEdgeIndex = new int[_n];
            Array.Fill(dist, int.MaxValue);
            Array.Fill(parent, -1);
            
            dist[s] = 0;
            var pq = new PriorityQueue<int, int>();
            pq.Enqueue(s, 0);
            
            while (pq.Count > 0)
            {
                int u = pq.Dequeue();
                
                if (dist[u] < int.MaxValue / 2) // Защита от переполнения
                {
                    for (int i = 0; i < _graph[u].Count; i++)
                    {
                        var edge = _graph[u][i];
                        if (edge.Capacity - edge.Flow > 0)
                        {
                            int newCost = dist[u] + edge.Cost + h[u] - h[edge.To];
                            if (newCost < dist[edge.To])
                            {
                                dist[edge.To] = newCost;
                                parent[edge.To] = u;
                                parentEdgeIndex[edge.To] = i;
                                pq.Enqueue(edge.To, dist[edge.To]);
                            }
                        }
                    }
                }
            }
            
            if (dist[t] == int.MaxValue)
                break; // Пути нет
            
            // Обновляем потенциалы
            for (int i = 0; i < _n; i++)
            {
                if (dist[i] != int.MaxValue)
                    h[i] += dist[i];
            }
            
            // Увеличиваем поток
            int push = requiredFlow - flow;
            int cur = t;
            while (cur != s)
            {
                int prev = parent[cur];
                var edge = _graph[prev][parentEdgeIndex[cur]];
                push = Math.Min(push, edge.Capacity - edge.Flow);
                cur = prev;
            }
            
            cur = t;
            while (cur != s)
            {
                int prev = parent[cur];
                var edge = _graph[prev][parentEdgeIndex[cur]];
                edge.Flow += push;
                _graph[cur][edge.ReverseEdgeIndex].Flow -= push;
                totalCost += push * edge.Cost;
                cur = prev;
            }
            
            flow += push;
        }
        
        return (flow, totalCost);
    }
}
```

**Пример сети, где без потенциалов Дейкстра неверна:**

Рассмотрим сеть:
- $s \to a$ с $c=1, w=0$.
- $a \to t$ с $c=1, w=-1$.
- $s \to t$ с $c=1, w=1$.

Без потенциалов Дейкстра не пойдёт по ребру с отрицательным весом $a \to t$ и выберет путь $s \to t$ со стоимостью 1, хотя оптимальный путь $s \to a \to t$ имеет стоимость $-1$.

С потенциалами после первой итерации веса пересчитываются, и отрицательные рёбра становятся неотрицательными.

### 13. Выбор алгоритма для поиска множества подстрок

**Условие:** Текст $t$ фиксирован и очень большой, образцы $p$ поступают онлайн. Какой подход выбрать: суффиксный массив или Ахо-Корасик?

**Решение:**

**Сравнение подходов:**

| Критерий | Суффиксный массив | Ахо-Корасик |
|----------|-------------------|-------------|
| Предобработка текста | $O(|t| \log |t|)$ или $O(|t|)$ | Не требуется |
| Предобработка образцов | Не требуется | $O(\sum |p_i|)$ на построение автомата |
| Память | $O(|t|)$ | $O(\sum |p_i|)$ |
| Время запроса | $O(|p| \cdot \log |t|)$ или $O(|p| + \log |t|)$ | $O(|t| + k)$ на весь текст |
| Добавление новых образцов | Мгновенно (не влияет) | Требуется перестройка автомата |

**Рекомендация:**

Для условия задачи (фиксированный текст, много онлайн-запросов) **суффиксный массив предпочтительнее**:

1. **Предобработка текста** выполняется один раз за $O(|t| \log |t|)$.
2. **Каждый запрос** обрабатывается за $O(|p| \cdot \log |t|)$ с бинарным поиском по суффиксному массиву или за $O(|p| + \log |t| + k)$ с LCP-массивом и бинарным поиском.
3. **Новые образцы** не требуют перестройки структуры.

Ахо-Корасик лучше подходит, когда:
- Образцы известны заранее (оффлайн-режим).
- Нужно найти все вхождения всех образцов за один проход по тексту.
- Текст поступает онлайн, а образцы фиксированы.

**Пример реализации поиска с суффиксным массивом:**

```csharp
public class SuffixArraySearch
{
    private readonly string _text;
    private readonly int[] _suffixArray;
    
    public SuffixArraySearch(string text)
    {
        _text = text;
        _suffixArray = BuildSuffixArray(text);
    }
    
    public List<int> FindAllOccurrences(string pattern)
    {
        var result = new List<int>();
        
        // Бинарный поиск левой границы
        int left = BinarySearchLeft(pattern);
        if (left == -1)
            return result;
        
        // Бинарный поиск правой границы
        int right = BinarySearchRight(pattern);
        
        for (int i = left; i <= right; i++)
            result.Add(_suffixArray[i]);
        
        return result;
    }
    
    private int BinarySearchLeft(string pattern)
    {
        int lo = 0, hi = _suffixArray.Length - 1;
        int result = -1;
        
        while (lo <= hi)
        {
            int mid = (lo + hi) / 2;
            int pos = _suffixArray[mid];
            int cmp = CompareSubstringWithPattern(pos, pattern);
            
            if (cmp >= 0)
            {
                if (cmp == 0) result = mid;
                hi = mid - 1;
            }
            else
            {
                lo = mid + 1;
            }
        }
        
        return result;
    }
    
    private int CompareSubstringWithPattern(int startPos, string pattern)
    {
        for (int i = 0; i < pattern.Length; i++)
        {
            if (startPos + i >= _text.Length)
                return -1; // Подстрока короче паттерна
            
            int cmp = _text[startPos + i].CompareTo(pattern[i]);
            if (cmp != 0)
                return cmp;
        }
        
        return 0; // Паттерн совпал или является префиксом подстроки
    }
    
    private int[] BuildSuffixArray(string s)
    {
        // Реализация за O(n log^2 n)
        int n = s.Length;
        var suffixes = new (string suffix, int index)[n];
        
        for (int i = 0; i < n; i++)
            suffixes[i] = (s.Substring(i), i);
        
        Array.Sort(suffixes, (a, b) => string.Compare(a.suffix, b.suffix, StringComparison.Ordinal));
        
        var sa = new int[n];
        for (int i = 0; i < n; i++)
            sa[i] = suffixes[i].index;
        
        return sa;
    }
}
```

```csharp
public class MyLinkedList<T> : IEquatable<MyLinkedList<T>>
{
    private class Node
    {
        public T Value;
        public Node? Prev;
        public Node? Next;
        
        public Node(T value) => Value = value;
    }
    
    private Node? _head;
    private Node? _tail;
    private int _count;
    
    public int Count => _count;
    
    public void AddLast(T value)
    {
        var newNode = new Node(value);
        if (_tail is null)
        {
            _head = _tail = newNode;
        }
        else
        {
            newNode.Prev = _tail;
            _tail.Next = newNode;
            _tail = newNode;
        }
        _count++;
    }
    
    public void AddFirst(T value)
    {
        var newNode = new Node(value);
        if (_head is null)
        {
            _head = _tail = newNode;
        }
        else
        {
            newNode.Next = _head;
            _head.Prev = newNode;
            _head = newNode;
        }
        _count++;
    }
    
    public bool Remove(T value)
    {
        var current = _head;
        while (current is not null)
        {
            if (EqualityComparer<T>.Default.Equals(current.Value, value))
            {
                if (current.Prev is not null)
                    current.Prev.Next = current.Next;
                else
                    _head = current.Next;
                
                if (current.Next is not null)
                    current.Next.Prev = current.Prev;
                else
                    _tail = current.Prev;
                
                _count--;
                return true;
            }
            current = current.Next;
        }
        return false;
    }

                for (int i = 0; i < index; i++)
                    current = current!.Next;
            }
            else
            {
                current = _tail;
                for (int i = _count - 1; i > index; i--)
                    current = current!.Prev;
            }
            
            return current!.Value;
        }
    }
```

```csharp
            _head = 0;

        _head = (_head + 1) % _array.Length;
        _count--;

        {
            var newArray = new T[_array.Length / 2];
            CopyTo(newArray, 0);
            _array = newArray;
            _head = 0;
            _tail = _count;
        }
```

```csharp
public class BinaryHeap<T> : IEquatable<BinaryHeap<T>>
{
    private readonly List<T> _elements;
    private readonly IComparer<T> _comparer;
    private readonly bool _isMinHeap;
    
    public int Count => _elements.Count;
    
    public BinaryHeap(bool isMinHeap = true, IComparer<T>? comparer = null)
    {
        _elements = new List<T>();
        _isMinHeap = isMinHeap;
        _comparer = comparer ?? Comparer<T>.Default;
    }
    
    public void Insert(T item)
    {
        _elements.Add(item);
        int index = _elements.Count - 1;
        
        while (index > 0)
        {
            int parentIndex = (index - 1) / 2;
            if (Compare(_elements[index], _elements[parentIndex]) < 0)
            {
                Swap(index, parentIndex);
                index = parentIndex;
            }
            else
            {
                break;
            }
        }
    }
    
    public T Peek()
    {
        if (_elements.Count == 0)
            throw new InvalidOperationException("Heap is empty");
        
        return _elements[0];
    }
    
    public T Extract()
    {
        if (_elements.Count == 0)
            throw new InvalidOperationException("Heap is empty");
        
        T result = _elements[0];
        int lastIndex = _elements.Count - 1;
        _elements[0] = _elements[lastIndex];
        _elements.RemoveAt(lastIndex);
        
        if (_elements.Count > 0)
        {
            HeapifyDown(0);
        }
        
        return result;
    }
    
    public void ChangeKey(int index, T newItem)
    {
        if (index < 0 || index >= _elements.Count)
            throw new ArgumentOutOfRangeException(nameof(index));
        
        T oldItem = _elements[index];
        _elements[index] = newItem;
        
        int comparison = Compare(newItem, oldItem);
        if (comparison < 0)
        {
            HeapifyUp(index);
        }
        else if (comparison > 0)
        {
            HeapifyDown(index);
        }
    }
    
    public T this[int index]
    {
        get
        {
            if (index < 0 || index >= _elements.Count)
                throw new ArgumentOutOfRangeException(nameof(index));
            return _elements[index];
        }
    }
    
    private void HeapifyUp(int index)
    {
        while (index > 0)
        {
            int parentIndex = (index - 1) / 2;
            if (Compare(_elements[index], _elements[parentIndex]) < 0)
            {
                Swap(index, parentIndex);
                index = parentIndex;
            }
            else
            {
                break;
            }
        }
    }
    
    private void HeapifyDown(int index)
    {
        int count = _elements.Count;
        
        while (true)
        {
            int smallest = index;
            int leftChild = 2 * index + 1;
            int rightChild = 2 * index + 2;
            
            if (leftChild < count && Compare(_elements[leftChild], _elements[smallest]) < 0)
            {
                smallest = leftChild;
            }
            
            if (rightChild < count && Compare(_elements[rightChild], _elements[smallest]) < 0)
            {
                smallest = rightChild;
            }
            
            if (smallest != index)
            {
                Swap(index, smallest);
                index = smallest;
            }
            else
            {
                break;
            }
        }
    }
    
    private void Swap(int i, int j)
    {
        T temp = _elements[i];
        _elements[i] = _elements[j];
        _elements[j] = temp;
    }
    
    private int Compare(T a, T b)
    {
        int result = _comparer.Compare(a, b);
        return _isMinHeap ? result : -result;
    }
    
    public override bool Equals(object? obj) => Equals(obj as BinaryHeap<T>);
    
    public bool Equals(BinaryHeap<T>? other)
    {
        if (other is null) return false;
        if (_elements.Count != other._elements.Count) return false;
        
        // Сравниваем как мультимножества
        var thisSorted = _elements.OrderBy(x => x).ToList();
        var otherSorted = other._elements.OrderBy(x => x).ToList();
        
        for (int i = 0; i < thisSorted.Count; i++)
        {
            if (!EqualityComparer<T>.Default.Equals(thisSorted[i], otherSorted[i]))
                return false;
        }
        
        return true;
    }
    
    public override int GetHashCode()
    {
        int hash = 17;
        foreach (var item in _elements.OrderBy(x => x))
        {
            hash = hash * 31 + (EqualityComparer<T>.Default.GetHashCode(item) ?? 0);
        }
        return hash;
    }
    
    public override string ToString()
    {
        return $"BinaryHeap [{string.Join(", ", _elements)}]";
    }
    
    public static BinaryHeap<T> operator +(BinaryHeap<T> left, BinaryHeap<T> right)
    {
        if (left is null) return right!;
        if (right is null) return left;
        
        var result = new BinaryHeap<T>(left._isMinHeap, left._comparer);
        
        foreach (var item in left._elements)
            result.Insert(item);
        
        foreach (var item in right._elements)
            result.Insert(item);
        
        return result;
    }
    
    public static bool operator ==(BinaryHeap<T>? left, BinaryHeap<T>? right)
    {
        if (left is null) return right is null;
        return left.Equals(right);
    }
    
    public static bool operator !=(BinaryHeap<T>? left, BinaryHeap<T>? right) => !(left == right);
}
```

### 7. Класс HashTable

```csharp
public class HashTable : IEquatable<HashTable>
{
    private class Node
    {
        public string Key;
        public int Value;
        public Node? Next;
        
        public Node(string key, int value)
        {
            Key = key;
            Value = value;
        }
    }
    
    private readonly List<Node?> _buckets;
    private int _count;
    
    public int Count => _count;
    public int Capacity => _buckets.Count;
    
    public HashTable(int capacity = 16)
    {
        _buckets = new List<Node?>(new Node?[capacity]);
        _count = 0;
    }
    
    private int GetBucketIndex(string key)
    {
        int hash = key.GetHashCode();
        return Math.Abs(hash) % _buckets.Count;
    }
    
    public void AddOrUpdate(string key, int value)
    {
        int index = GetBucketIndex(key);
        Node? current = _buckets[index];
        
        while (current is not null)
        {
            if (current.Key == key)
            {
                current.Value = value;
                return;
            }
            current = current.Next;
        }
        
        var newNode = new Node(key, value);
        newNode.Next = _buckets[index];
        _buckets[index] = newNode;
        _count++;
        
        // Перехеширование при заполнении > 75%
        if (_count > _buckets.Count * 0.75)
        {
            Resize();
        }
    }
    
    public bool TryGetValue(string key, out int value)
    {
        value = 0;
        int index = GetBucketIndex(key);
        Node? current = _buckets[index];
        
        while (current is not null)
        {
            if (current.Key == key)
            {
                value = current.Value;
                return true;
            }
            current = current.Next;
        }
        
        return false;
    }
    
    public bool Remove(string key)
    {
        int index = GetBucketIndex(key);
        Node? current = _buckets[index];
        Node? prev = null;
        
        while (current is not null)
        {
            if (current.Key == key)
            {
                if (prev is null)
                {
                    _buckets[index] = current.Next;
                }
                else
                {
                    prev.Next = current.Next;
                }
                _count--;
                return true;
            }
            prev = current;
            current = current.Next;
        }
        
        return false;
    }
    
    public int this[string key]
    {
        get
        {
            if (TryGetValue(key, out int value))
                return value;
            throw new KeyNotFoundException($"Key '{key}' not found");
        }
        set => AddOrUpdate(key, value);
    }
    
    public IEnumerable<string> Keys
    {
        get
        {
            foreach (var bucket in _buckets)
            {
                Node? current = bucket;
                while (current is not null)
                {
                    yield return current.Key;
                    current = current.Next;
                }
            }
        }
    }
    
    private void Resize()
    {
        var oldBuckets = _buckets;
        _buckets.Clear();
        _buckets.Capacity = oldBuckets.Count * 2;
        for (int i = 0; i < _buckets.Capacity; i++)
            _buckets.Add(null);
        
        _count = 0;
        
        foreach (var bucket in oldBuckets)
        {
            Node? current = bucket;
            while (current is not null)
            {
                AddOrUpdate(current.Key, current.Value);
                current = current.Next;
            }
        }
    }
    
    public override bool Equals(object? obj) => Equals(obj as HashTable);
    
    public bool Equals(HashTable? other)
    {
        if (other is null) return false;
        if (_count != other._count) return false;
        
        foreach (var key in Keys)
        {
            if (!TryGetValue(key, out int thisValue) ||
                !other.TryGetValue(key, out int otherValue) ||
                thisValue != otherValue)
            {
                return false;
            }
        }
        
        return true;
    }
    
    public override int GetHashCode()
    {
        int hash = 0;
        foreach (var key in Keys.OrderBy(k => k))
        {
            TryGetValue(key, out int value);
            hash = hash * 31 + (key.GetHashCode() ^ value);
        }
        return hash;
    }
    
    public override string ToString()
    {
        var sb = new StringBuilder();
        sb.Append("HashTable {");
        bool first = true;
        foreach (var key in Keys)
        {
            if (!first) sb.Append(", ");
            TryGetValue(key, out int value);
            sb.Append($"\"{key}\": {value}");
            first = false;
        }
        sb.Append('}');
        return sb.ToString();
    }
    
    public static bool operator ==(HashTable? left, HashTable? right)
    {
        if (left is null) return right is null;
        return left.Equals(right);
    }
    
    public static bool operator !=(HashTable? left, HashTable? right) => !(left == right);
}
```
