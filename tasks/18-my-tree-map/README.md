# MyTreeMap — Документация разработчика

## 📌 Обзор

`MyTreeMap<K, V>` — это реализация **отображения на основе бинарного дерева поиска (BST)**. 
Класс предоставляет функционал, аналогичный `TreeMap` в Java или `std::map` в C++.

> ⚠️ **Важно**: Дерево **не сбалансировано** (не AVL, не красно-чёрное). При вставке 
> отсортированных данных вырождается в связный список → O(n) вместо O(log n).

### Основные характеристики

| Параметр | Значение |
|----------|----------|
| **Структура** | Бинарное дерево поиска (BST) **без балансировки** |
| **Упорядочивание** | Ключи сортируются в порядке возрастания |
| **Сложность операций** | **O(log n) в среднем**, **O(n) в худшем случае** |
| **Допустимые ключи** | Любые типы, реализующие `IComparable<K>` или с кастомным `IComparer<K>` |
| **Null-ключи** | ❌ Запрещены (выбрасывается `ArgumentNullException`) |

---

## 🏗️ Архитектура класса

### Поля класса

```csharp
private Node? root;           // Корень дерева
private int size;             // Количество элементов
private IComparer<K>? comparator;  // Опциональный кастомный компаратор
```

### Вложенный класс Node

```csharp
private class Node
{
    public K key;             // Ключ узла
    public V value;           // Значение узла
    public Node? left;        // Левый потомок (ключи < текущего)
    public Node? right;       // Правый потомок (ключи > текущего)
}
```

**Инвариант дерева**: для любого узла все ключи в левом поддереве **меньше**, а в правом — **больше** текущего ключа.

## 🔧 Конструкторы

### 1. Конструктор по умолчанию
```csharp
public MyTreeMap()
```
Использует естественный порядок ключей (через `IComparable<K>`).

### 2. Конструктор с кастомным компаратором
```csharp
public MyTreeMap(IComparer<K> comp)
```
Позволяет задать свой порядок сортировки ключей.

**Пример** (обратный порядок строк):
```csharp
var map = new MyTreeMap<string, int>(
    Comparer<string>.Create((a, b) => b.CompareTo(a))
);
```

## 📚 API Reference

### Базовые операции

| Метод/Свойство | Описание | Сложность |
|----------------|----------|-----------|
| `Size` (свойство) | Количество элементов | O(1) |
| `IsEmpty()` | Проверка на пустоту | O(1) |
| `Clear()` | Очистка дерева | O(1) |
| `Put(key, value)` | Добавить/обновить пару | O(log n) |
| `Get(key)` | Получить значение по ключу | O(log n) |
| `this[key]` (индексатор) | Альтернативный доступ через `map[key]` | O(log n) |
| `ContainsKey(key)` | Проверка наличия ключа | O(log n) |
| `ContainsValue(value)` | Проверка наличия значения | O(n) |
| `Remove(key)` | Удаление по ключу | O(log n) |

---

### Навигация по ключам

| Метод | Описание | Пример |
|-------|----------|--------|
| `FirstKey()` | Минимальный ключ | Для `{1, 5, 10}` → `1` |
| `LastKey()` | Максимальный ключ | Для `{1, 5, 10}` → `10` |
| `FirstEntry()` | Первая пара (ключ, значение) | `(1, "A")` |
| `LastEntry()` | Последняя пара | `(10, "C")` |
| `PollFirstEntry()` | Получить и удалить первую | Удаляет из дерева |
| `PollLastEntry()` | Получить и удалить последнюю | Удаляет из дерева |

---

### Поиск ближайших ключей

| Метод | Описание | Возвращает |
|-------|----------|------------|
| `LowerKey(key)` | Наибольший ключ **строго меньше** `key` | Для `key=5` в `{1, 3, 7}` → `3` |
| `FloorKey(key)` | Наибольший ключ **≤** `key` | Для `key=5` в `{1, 5, 7}` → `5` |
| `HigherKey(key)` | Наименьший ключ **строго больше** `key` | Для `key=5` в `{1, 3, 7}` → `7` |
| `CeilingKey(key)` | Наименьший ключ **≥** `key` | Для `key=5` в `{1, 3, 7}` → `7` |

Аналогичные методы для пар **Entry**: `LowerEntry`, `FloorEntry`, `HigherEntry`, `CeilingEntry`.

---

### Представления (Views)

| Метод | Описание | Возвращает |
|-------|----------|------------|
| `KeySet()` | Все ключи в порядке возрастания | `HashSet<K>` |
| `EntrySet()` | Все пары (ключ, значение) | `HashSet<KeyValuePair<K, V>>` |

---

### Под-карты (Submaps)

| Метод | Описание | Диапазон |
|-------|----------|----------|
| `HeadMap(end)` | Ключи **< end** | `(-∞, end)` |
| `TailMap(start)` | Ключи **> start** | `(start, +∞)` |
| `SubMap(start, end)` | Ключи в **[start, end)** | `[start, end)` |

**Важно**: Методы возвращают **новую копию** `MyTreeMap`, а не live-view.

## 🧠 Алгоритмы ключевых методов

### Put (Вставка)

```
1. Если дерево пустое → создать корень
2. Иначе рекурсивно спускаться:
   - Если ключ < текущего → влево
   - Если ключ > текущего → вправо
   - Если ключ == текущему → обновить значение
3. Увеличить size при создании нового узла
```

**Реализация**: рекурсивная через `PutNode`.

---

### Get (Поиск)

```
1. Начать с корня
2. Сравнивать искомый ключ с текущим:
   - Если < → идти влево
   - Если > → идти вправо
   - Если == → вернуть значение
3. Если достигли null → ключ не найден
```

**Реализация**: итеративная через `GetNode`.

---

### Remove (Удаление)

Три случая:

1. **Узел без потомков** → просто удалить
2. **Узел с одним потомком** → заменить узлом потомка
3. **Узел с двумя потомками** → найти **наименьший в правом поддереве** (successor), скопировать его ключ/значение, рекурсивно удалить successor

```
     10           Удалить 10:
    /  \          1. Найти min в правом: 15
   5   20         2. Заменить 10 → 15
      /  \        3. Удалить старый узел 15
     15  25
```

---

### HeadMap / TailMap / SubMap

**Общий принцип**: обход дерева с фильтрацией по диапазону.

Пример для `HeadMap(end)`:
```csharp
if (node.key < end)
{
    // Обработать левое поддерево (все ключи < node.key < end)
    HeadMapCollect(node.left, map, end);
    // Добавить текущий узел
    map.Put(node.key, node.value);
    // Обработать правое поддерево (может содержать ключи < end)
    HeadMapCollect(node.right, map, end);
}
else
{
    // Все ключи в правом поддереве >= end, обрабатываем только левое
    HeadMapCollect(node.left, map, end);
}
```

---

### Lower/Floor/Higher/Ceiling

**Единый паттерн**: итеративный спуск с запоминанием кандидата.

Пример для `FloorKey(key)` (наибольший ключ ≤ key):
```csharp
Node? result = null;
while (node != null)
{
    int cmp = Compare(node.key, key);
    if (cmp == 0) return node;        // Точное совпадение
    if (cmp < 0)                      // node.key < key → кандидат
    { 
        result = node; 
        node = node.right;            // Искать больше
    }
    else 
    { 
        node = node.left;             // Искать меньше
    }
}
return result;
```

## 🔒 Обработка ошибок

| Ситуация | Исключение |
|----------|------------|
| Передача `null` ключа | `ArgumentNullException` |
| `FirstKey()` / `LastKey()` на пустой карте | `InvalidOperationException` |
| `SubMap(start, end)` где `start >= end` | `ArgumentException` |

## ⚠️ Ограничения и известные проблемы

### 1. Отсутствие балансировки ⚠️ КРИТИЧНО

Дерево **не является сбалансированным** (не AVL, не красно-чёрное). 

**Последствия**:
- При вставке **отсортированных** или **почти отсортированных** данных дерево вырождается в связный список
- Сложность операций деградирует до **O(n)** вместо **O(log n)**

**Пример**:
```csharp
var map = new MyTreeMap<int, string>();
for (int i = 1; i <= 10000; i++)
    map.Put(i, $"V{i}");  // Вырожденное дерево, глубина = 10000
```

**Решение**: 
- Для продакшена использовать `SortedDictionary<K, V>` (красно-чёрное дерево)
- Реализовать балансировку (TODO)

### 2. SubMap не является live-view
Методы `HeadMap`, `TailMap`, `SubMap` возвращают **копию**, а не представление. Изменения в исходной карте не отражаются в под-карте.

### 3. EntrySet/KeySet возвращают HashSet
Порядок элементов в `HashSet` **не гарантирован**, хотя обход дерева идёт в отсортированном порядке.

## 📖 Примеры использования

### Базовое использование
```csharp
var map = new MyTreeMap<string, int>();
map.Put("apple", 1);
map.Put("banana", 2);
map.Put("cherry", 3);

// Через индексатор
map["date"] = 4;
int value = map["banana"];  // 2

// Навигация
var first = map.FirstKey();      // "apple"
var last = map.LastKey();        // "date"
var higher = map.HigherKey("banana");  // "cherry"
```

### Кастомный компаратор
```csharp
// Обратный порядок
var reverseMap = new MyTreeMap<int, string>(
    Comparer<int>.Create((a, b) => b.CompareTo(a))
);
reverseMap.Put(1, "one");
reverseMap.Put(2, "two");
// Первый ключ будет 2, последний — 1
```

### Работа с диапазонами
```csharp
var map = new MyTreeMap<int, string>();
for (int i = 1; i <= 10; i++)
    map.Put(i, $"V{i}");

var subMap = map.SubMap(3, 7);  // {3=V3, 4=V4, 5=V5, 6=V6}
var headMap = map.HeadMap(5);   // {1=V1, 2=V2, 3=V3, 4=V4}
var tailMap = map.TailMap(8);   // {9=V9, 10=V10}
```

## 🎯 Рекомендации для разработчика

### При чтении кода обращайте внимание на:

1. **Метод `Compare`** — центральный элемент, определяющий порядок:
   ```csharp
   private int Compare(K key1, K key2)
   {
       return comparator?.Compare(key1, key2) ?? key1.CompareTo(key2);
   }
   ```

2. **Рекурсивные методы** (`PutNode`, `RemoveNode`) — возвращают новый корень поддерева для обновления ссылок.

3. **Итеративные методы** (`GetNode`, `FindFloorNode`) — более эффективны, не используют стек вызовов.

### При расширении функционала:

- Добавляйте проверку на `null` для ключей в начале публичных методов
- Сохраняйте инвариант BST при любых модификациях
- Обновляйте `size` при добавлении/удалении узлов
- Для методов навигации обрабатывайте случай пустого дерева
