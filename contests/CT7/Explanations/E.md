# Компоненты вершинной двусвязности

## Условие задачи
Компонента вершинной двусвязности — максимальный по включению подграф, где любые два ребра лежат на вершинно простом цикле. Требуется разбить рёбра графа на такие компоненты.

## Алгоритм
Используется **DFS со стеком рёбер** для выделения блоков (компонент).

### Идея алгоритма
Компоненты вершинной двусвязности (также называемые **блоками**) — это максимальные двусвязные подграфы. Рёбра разбиваются на блоки, где каждый блок — это либо цикл, либо отдельное ребро-мост.

Ключевое наблюдение: при DFS рёбра, посещённые между моментом входа в вершину u и моментом, когда `low[v] >= tin[u]`, образуют одну компоненту.

### Структуры данных
- `adj[]` — списки смежности с индексами рёбер
- `tin[]`, `low[]` — время захода и минимальное достижимое время
- `edgeComponent[]` — номер компоненты для каждого ребра
- `edgeStack` — стек рёбер в текущем пути DFS
- `componentCount` — количество компонент

### Ход алгоритма
1. **DFS обход**:
   - Устанавливаем `tin[u] = low[u] = ++timer`
   - Для каждого соседа v:
     - Если v — родитель → пропускаем
     - Если v уже посещена:
       - `low[u] = min(low[u], tin[v])`
       - Если `tin[v] < tin[u]` → добавляем ребро в стек (обратное ребро вверх)
     - Если v не посещена:
       - Добавляем ребро (u, v) в стек
       - DFS(v)
       - `low[u] = min(low[u], low[v])`
       - Если `low[v] >= tin[u]`:
         - Вынимаем рёбра из стека до (u, v) включительно
         - Всем вынутым рёбрам присваиваем `componentCount++`

2. **Вывод**: номера компонент для рёбер 1..m

### Пример работы
```
Вход:
5 6
1-2, 2-3, 3-1, 1-4, 4-5, 5-1

Граф:
    1
   /|\
  2-3 4
     / \
    5---/

Компоненты:
- Компонента 1: рёбра {1-2, 2-3, 3-1} (треугольник)
- Компонента 2: рёбра {1-4, 4-5, 5-1} (треугольник)

Вывод:
2
1 1 1 2 2 2
```

### Сложность
- **Время**: O(n + m) — один проход DFS
- **Память**: O(n + m) — граф, массивы, стек

### Ключевые моменты реализации
1. **Стек рёбер**: храним рёбра, а не вершины
2. **Условие выделения**: `low[v] >= tin[u]` — выделена полная компонента
3. **Обратные рёбра**: добавляем в стек только обратные рёбра, идущие вверх по дереву
4. **Нумерация рёбер**: рёбра нумеруются в порядке входа во входном файле
