# E. Компоненты вершинной двусвязности

## Введение

Задача на выделение **компонент вершинной двусвязности** в неориентированном графе. Компонента вершинной двусвязности — это максимальный по включению подграф, в котором любые два ребра лежат на простом цикле.

Решение использует **DFS со стеком рёбер**: при обнаружении точки сочленения все рёбра в стеке формируют новую компоненту.

## Идея

В отличие от рёберной двусвязности, здесь компоненты определяются **рёбрами**, а не вершинами. Ключевое наблюдение:

- При обходе DFS рёбра добавляются в стек
- Когда обнаруживается точка сочленения (`low[v] >= tin[u]`), все рёбра до текущего образуют компоненту вершинной двусвязности

Важно: одно ребро может принадлежать только одной компоненте.

## Разбор кода

### Инициализация

```csharp
adj = new List<(int, int)>[n + 1];
for (int i = 1; i <= n; i++)
    adj[i] = new List<(int, int)>();
```

Список смежности с индексами рёбер для нумерации ответа.

```csharp
tin = new int[n + 1];
low = new int[n + 1];
edgeComponent = new int[m + 1];
edgeStack = new Stack<int>();
timer = 0;
componentCount = 0;
```

- `tin`, `low` — времена входа и возврата
- `edgeComponent` — номер компоненты для каждого ребра
- `edgeStack` — стек рёбер текущего пути

### Обход графа

```csharp
for (int i = 1; i <= n; i++)
{
    if (tin[i] == 0)
        DFS(i, -1);
}
```

DFS из каждой непосещённой вершины (граф может быть несвязным).

### Функция DFS

```csharp
private static void DFS(int u, int parentEdgeIndex)
{
    tin[u] = low[u] = ++timer;
```

Инициализация времён для новой вершины.

```csharp
    foreach (var (v, edgeIndex) in adj[u])
    {
        if (edgeIndex == parentEdgeIndex)
            continue;
```

Пропускаем ребро, по которому пришли. Проверка по **индексу ребра** корректно обрабатывает кратные рёбра.

```csharp
        if (tin[v] != 0)
        {
            low[u] = Math.Min(low[u], tin[v]);
            if (tin[v] < tin[u])
                edgeStack.Push(edgeIndex);
        }
```

Для обратного ребра:
- Обновляем `low[u]` через `tin[v]`
- Добавляем ребро в стек, если оно ведёт к предку (условие `tin[v] < tin[u]`)

```csharp
        else
        {
            edgeStack.Push(edgeIndex);
            DFS(v, edgeIndex);
            low[u] = Math.Min(low[u], low[v]);
```

Для дерева DFS:
- Сначала добавляем ребро в стек
- Рекурсивно обрабатываем поддерево
- Обновляем `low[u]` после возврата

```csharp
            if (low[v] >= tin[u])
            {
                componentCount++;
                while (true)
                {
                    int edge = edgeStack.Pop();
                    edgeComponent[edge] = componentCount;
                    if (edge == edgeIndex)
                        break;
                }
            }
        }
    }
}
```

**Ключевой момент**: когда `low[v] >= tin[u]`, вершина `u` — точка сочленения. Все рёбра в стеке до текущего ребра `(u, v)` образуют новую компоненту вершинной двусвязности.

### Вывод результата

```csharp
Console.WriteLine(componentCount);
var result = new int[m];
for (int i = 1; i <= m; i++)
    result[i - 1] = edgeComponent[i];
Console.WriteLine(string.Join(" ", result));
```

Выводим количество компонент и номер компоненты для каждого ребра в порядке входа.

## Оптимизации

- **Стек рёбер**: эффективное выделение компонент за O(1) на ребро
- **Однопроходный DFS**: компоненты находятся во время обхода
- **Временная сложность**: O(n + m)
- **Пространственная сложность**: O(n + m)

## Резюме

Алгоритм выделяет компоненты вершинной двусвязности за один проход DFS. Рёбра добавляются в стек при первом посещении, а при обнаружении точки сочленения извлекаются и получают номер новой компоненты. Каждое ребро принадлежит ровно одной компоненте.
