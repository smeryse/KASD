# Максимальный поток (Форд-Фалкерсон)

## Условие задачи
Дан ориентированный граф с пропускными способностями рёбер. Найти максимальный поток от истока `s` к стоку `t`.

## Алгоритм
Используется **алгоритм Форда-Фалкерсона** с поиском дополняющих путей через BFS (разновидность — алгоритм Эдмондса-Карпа).

### Идея алгоритма
1. Начинаем с нулевого потока
2. Ищем любой путь от `s` к `t` с положительной остаточной пропускной способностью
3. Увеличиваем поток вдоль этого пути на минимальную остаточную способность
4. Повторяем, пока существуют дополняющие пути

**Остаточный граф**: для каждого ребра (u, v) с потоком f и пропускной способностью c:
- Прямое ребро: остаточная способность = c - f
- Обратное ребро: остаточная способность = f (можно "отменить" поток)

### Структуры данных
- `adj[]` — списки смежности, хранят (to, cap, flow, rev):
  - `to` — конечная вершина
  - `cap` — пропускная способность
  - `flow` — текущий поток
  - `rev` — индекс обратного ребра в списке смежности `to`
- `visited[]` — для поиска пути
- `parent[]` — для восстановления пути (откуда пришли и каким ребром)

### Ход алгоритма
1. **Инициализация**: читаем граф, создаём обратные рёбра с нулевой пропускной способностью

2. **Поиск дополняющих путей** (внешний цикл):
   - BFS от `s` к `t` по рёбрам с `cap - flow > 0`
   - Если путь найден:
     - Находим минимальную остаточную способность `push` вдоль пути
     - Увеличиваем поток на `push` (внутренний цикл)
     - Обновляем потоки в прямых и обратных рёбрах
   - Если пути нет → завершаем

3. **Вывод**: суммарный поток из `s`

### Пример работы
```
Вход:
4 5
1→2 (10), 1→3 (10), 2→3 (5), 2→4 (5), 3→4 (10)
s=1, t=4

Граф:
      10
    1 → 2
    | ↙5| \ 5
  10|   ↓ |  → 4
    |   3 → 2
    ↓  10
    3

Итерация 1:
Путь: 1→2→4, push=min(10,5)=5
Поток: 5

Итерация 2:
Путь: 1→3→4, push=min(10,10)=10
Поток: 5+10=15

Итерация 3:
Путь: 1→2→3→4, push=min(10-5,5,10-10)=0
Нет пути с положительной способностью

Ответ: 15
```

### Сложность
- **Время**: O(V · E²) для Эдмондса-Карпа (BFS), но на практике быстрее
- **Память**: O(n + m) — граф с обратными рёбрами

### Ключевые моменты реализации
1. **Обратные рёбра**: необходимы для "отмены" потока
2. **Индекс обратного ребра**: `rev` позволяет быстро найти обратное ребро
3. **BFS для пути**: гарантирует кратчайший дополняющий путь (Эдмондс-Карп)
4. **Обновление потока**: увеличиваем в прямом, уменьшаем в обратном
