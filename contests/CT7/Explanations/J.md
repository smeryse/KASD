# Минимальное остовное дерево (Крускал)

## Условие задачи
Дан неориентированный взвешенный граф. Найти минимальное остовное дерево (MST) — подграф, соединяющий все вершины с минимальной суммой весов рёбер.

## Алгоритм
Используется **алгоритм Крускала** с системой непересекающихся множеств (DSU).

### Идея алгоритма
Жадный алгоритм: сортируем рёбра по весу и добавляем их в MST, если они соединяют разные компоненты связности.

**Система непересекающихся множеств (DSU)**:
- `parent[x]` — родитель вершины x
- `rank[x]` — ранг (приблизительная высота дерева)
- `Find(x)` — найти представителя множества с路径压缩
- `Union(x, y)` — объединить два множества по рангу

### Структуры данных
- `edges` — список рёбер (u, v, w), сортируется по весу
- `parent[]` — родитель в DSU
- `rank[]` — ранг в DSU
- `mstWeight` — сумма весов рёбер MST
- `edgesCount` — количество рёбер в MST

### Ход алгоритма
1. **Инициализация DSU**: каждая вершина — отдельное множество

2. **Сортировка рёбер**: по возрастанию веса

3. **Обход рёбер**:
   - Для каждого ребра (u, v, w):
     - `rootU = Find(u)`, `rootV = Find(v)`
     - Если `rootU != rootV`:
       - Добавляем ребро в MST: `mstWeight += w`
       - `edgesCount++`
       - `Union(rootU, rootV)`

4. **Проверка связности**: если `edgesCount < n - 1` и `n > 1` → граф несвязный

5. **Вывод**: вес MST или "No solution"

### Пример работы
```
Вход:
4 5
1-2 (1), 2-3 (2), 3-4 (3), 1-4 (4), 2-4 (5)

Сортировка рёбер:
(1-2, 1), (2-3, 2), (3-4, 3), (1-4, 4), (2-4, 5)

DSU: [{1}, {2}, {3}, {4}]

→ Ребро 1-2 (w=1): root1≠root2
  MST: {1-2}, weight=1
  DSU: [{1,2}, {3}, {4}]

→ Ребро 2-3 (w=2): root2≠root3
  MST: {1-2, 2-3}, weight=3
  DSU: [{1,2,3}, {4}]

→ Ребро 3-4 (w=3): root3≠root4
  MST: {1-2, 2-3, 3-4}, weight=6
  DSU: [{1,2,3,4}]

→ Ребро 1-4 (w=4): root1=root4 → пропускаем (цикл)
→ Ребро 2-4 (w=5): root2=root4 → пропускаем

edgesCount=3 = n-1 ✓

Ответ: 6
```

### Сложность
- **Время**: O(m log m) — сортировка рёбер, DSU операции почти O(1)
- **Память**: O(n + m) — рёбра и DSU

### Ключевые моменты реализации
1. **Сортировка по весу**: гарантирует жадный выбор минимального ребра
2. **DSU с оптимизациями**: сжатие путей и объединение по рангу
3. **Проверка цикла**: если вершины в одном множестве — ребро создаст цикл
4. **Несвязный граф**: проверяем количество рёбер в MST
