# Динамическое программирование на деревьях

## Условие задачи
Дано дерево. Найти максимальное независимое множество — наибольшее множество вершин, никакие две из которых не соединены ребром.

## Алгоритм
Используется **динамическое программирование на деревьях** (Tree DP) с DFS.

### Идея алгоритма
Для каждой вершины u вычисляем два значения:
- `dp0[u]` — размер максимального независимого множества в поддереве u, **если u не включена**
- `dp1[u]` — размер максимального независимого множества в поддереве u, **если u включена**

**Базовый случай** (лист):
- `dp0[leaf] = 0` (не включили лист)
- `dp1[leaf] = 1` (включили лист)

**Рекуррентные соотношения**:
- `dp1[u] = 1 + Σ dp0[v]` для всех детей v
  - Если включили u, то дети не могут быть включены
- `dp0[u] = Σ max(dp0[v], dp1[v])` для всех детей v
  - Если не включили u, то для каждого ребёнка выбираем лучший вариант

**Ответ**: `max(dp0[root], dp1[root])`

### Структуры данных
- `adj[]` — списки смежности
- `visited[]` — посещённые вершины (для предотвращения возврата к родителю)
- `dp0[]`, `dp1[]` — значения динамики

### Ход алгоритма
1. **DFS обход** от корня (вершина 1):
   - Помечаем вершину как посещённую
   - Инициализируем: `dp1[u] = 1`, `dp0[u] = 0`
   - Для каждого ребёнка v:
     - Рекурсивно вызываем DFS(v)
     - `dp1[u] += dp0[v]` (если взяли u, дети нельзя)
     - `dp0[u] += max(dp0[v], dp1[v])` (если не взяли u, дети — как лучше)

2. **Ответ**: `max(dp0[root], dp1[root])`

### Пример работы
```
Вход:
5 0
1-2, 1-3, 2-4, 2-5

Дерево:
      1
     / \
    2   3
   / \
  4   5

DFS(1):
  DFS(2):
    DFS(4): лист
      dp0[4]=0, dp1[4]=1
    DFS(5): лист
      dp0[5]=0, dp1[5]=1
    dp1[2] = 1 + dp0[4] + dp0[5] = 1 + 0 + 0 = 1
    dp0[2] = max(0,1) + max(0,1) = 1 + 1 = 2
  DFS(3): лист
    dp0[3]=0, dp1[3]=1
  dp1[1] = 1 + dp0[2] + dp0[3] = 1 + 2 + 0 = 3
  dp0[1] = max(2,1) + max(0,1) = 2 + 1 = 3

Ответ: max(3, 3) = 3

Варианты независимых множеств размера 3:
- {1, 4, 5}
- {2, 3}
- {3, 4, 5}
```

### Сложность
- **Время**: O(n) — один проход DFS, каждая вершина обрабатывается один раз
- **Память**: O(n) — граф и массивы динамики

### Ключевые моменты реализации
1. **Два состояния**: для каждой вершины считаем оба варианта (включена/не включена)
2. **Зависимость от детей**: dp1 зависит от dp0 детей, dp0 — от максимума
3. **Посещённые вершины**: используем visited вместо передачи родителя
4. **Ответ в корне**: максимум из двух состояний корня

### Обобщение
Этот подход применим ко многим задачам на деревьях:
- Максимальное независимое множество (данная задача)
- Минимальное вершинное покрытие
- Максимальное паросочетание
- Раскраска деревьев
