# Мосты

## Условие задачи
Дан неориентированный граф. Требуется найти все **мосты** — рёбра, удаление которых увеличивает количество компонент связности графа.

## Алгоритм
Используется алгоритм на основе **поиска в глубину (DFS)** с вычислением времени захода и точек возврата.

### Идея алгоритма
Ребро (u, v) является мостом тогда и только тогда, когда **не существует обратного ребра** из поддерева v в u или любого предка u. 

Ключевые понятия:
- **tin[u]** — время захода в вершину u (порядковый номер посещения в DFS)
- **low[u]** — минимальное время захода, достижимое из u по обратным рёбрам

Ребро (u, v) — мост ⇔ **low[v] > tin[u]**

### Структуры данных
- `adj[]` — списки смежности, хранят пары (сосед, индекс ребра)
- `tin[]` — время захода в вершину
- `low[]` — минимальное достижимое время захода
- `bridges` — список найденных мостов (индексы рёбер)
- `timer` — счётчик времени для нумерации вершин

### Ход алгоритма
1. **Инициализация**: читаем граф, сохраняем индексы рёбер
2. **DFS от каждой непосещённой вершины**:
   - Устанавливаем `tin[u] = low[u] = ++timer`
   - Для каждого соседа v:
     - Если v — родитель (то же ребро) → пропускаем
     - Если v уже посещена → обновляем `low[u] = min(low[u], tin[v])` (обратное ребро)
     - Если v не посещена:
       - Рекурсивно вызываем DFS(v)
       - Обновляем `low[u] = min(low[u], low[v])`
       - Если `low[v] > tin[u]` → ребро (u, v) — мост
3. **Вывод**: сортируем мосты по индексам, выводим количество и номера

### Пример работы
```
Вход:
6 7
1-2(1), 2-3(2), 3-4(3), 1-3(4), 4-5(5), 4-6(6), 5-6(7)

Граф:
    1
   / \
 (1) (4)
 /     \
2-------3
   (2)   \
         (3)
          \
           4
          / \
       (5) (6)
        /     \
       5-------6
         (7)

DFS(1):
tin[1]=1, low[1]=1
→ DFS(2): tin[2]=2, low[2]=2
  → DFS(3): tin[3]=3, low[3]=3
    → DFS(4): tin[4]=4, low[4]=4
      → DFS(5): tin[5]=5, low[5]=5
        → 6: tin[6]=6, low[6]=6
          → 5 (обратное): low[6]=min(6,5)=5
        → low[5]=min(5,5)=5
        → low[5]=5 > tin[4]=4? Нет (5>4) → ребро 5 не мост
      → DFS(6): уже посещено
      → low[4]=min(4,5)=4
      → low[4]=4 > tin[3]=3? Да → ребро 3 — МОСТ!
```

### Сложность
- **Время**: O(n + m) — один проход DFS
- **Память**: O(n + m) — граф и массивы

### Ключевые моменты реализации
1. **Индексы рёбер**: храним индекс каждого ребра для вывода ответа
2. **Игнорирование родителя**: не считаем обратным ребром переход к родителю
3. **Несвязный граф**: запускаем DFS от каждой непосещённой вершины
4. **Сортировка ответа**: мосты выводятся в возрастающем порядке номеров
