# B. Мосты

## Введение

Задача на нахождение всех **мостов** в неориентированном графе. Мост — это ребро, удаление которого увеличивает количество компонент связности графа.

Решение использует модификацию **поиска в глубину (DFS)** с вычислением времён входа `tin` и точек возврата `low`.

## Идея

Для каждой вершины вычисляем два значения:
- **`tin[u]`** (time in) — время первого посещения вершины `u` в DFS
- **`low[u]`** (low link) — минимальное `tin`, достижимое из `u` по дереву DFS и не более чем одному обратному ребру

Ребро `(u, v)` является мостом тогда и только тогда, когда `low[v] > tin[u]`. Это означает, что из поддерева `v` нельзя достичь `u` или её предков без использования ребра `(u, v)`.

## Разбор кода

### Инициализация

```csharp
adj = new List<(int, int)>[n + 1];
for (int i = 1; i <= n; i++)
    adj[i] = new List<(int, int)>();
```

Список смежности хранит пары `(вершина, индекс_ребра)`. Индекс ребра нужен для вывода ответа в правильном порядке.

```csharp
tin = new int[n + 1];
low = new int[n + 1];
bridges = new List<int>();
timer = 0;
```

- `tin` и `low` — массивы времён
- `bridges` — список найденных мостов (индексы рёбер)
- `timer` — глобальный счётчик времени

### Обход графа

```csharp
for (int i = 1; i <= n; i++)
{
    if (tin[i] == 0)
        DFS(i, -1);
}
```

Запускаем DFS из каждой непосещённой вершины, так как граф может быть несвязным. `-1` означает, что родительского ребра нет (корневая вершина).

### Функция DFS

```csharp
private static void DFS(int u, int parentEdge)
{
    tin[u] = low[u] = ++timer;
```

При первом посещении вершины устанавливаем оба значения в текущее время.

```csharp
    foreach (var (v, edgeIndex) in adj[u])
    {
        if (edgeIndex == parentEdge)
            continue;
```

Пропускаем ребро, по которому пришли в вершину. Важно проверять именно **индекс ребра**, а не вершину-родителя, чтобы корректно обрабатывать кратные рёбра.

```csharp
        if (tin[v] != 0)
        {
            low[u] = Math.Min(low[u], tin[v]);
        }
```

Если `v` уже посещена — это обратное ребро. Обновляем `low[u]` через `tin[v]` (не `low[v]`!).

```csharp
        else
        {
            DFS(v, edgeIndex);
            low[u] = Math.Min(low[u], low[v]);
            if (low[v] > tin[u])
                bridges.Add(edgeIndex);
        }
    }
}
```

Для непосещённой вершины:
1. Рекурсивно вызываем DFS
2. После возврата обновляем `low[u]` через `low[v]`
3. Проверяем условие моста: если `low[v] > tin[u]`, то ребро `(u, v)` — мост

### Вывод результата

```csharp
bridges.Sort();
Console.WriteLine(bridges.Count);
Console.WriteLine(string.Join(" ", bridges));
```

Сортируем мосты по индексам и выводим количество, затем сами индексы.

## Оптимизации

- **Однопроходный DFS**: мосты находятся во время обхода
- **Проверка индекса ребра**: корректная работа с кратными рёбрами
- **Временная сложность**: O(n + m log m) — DFS за O(n + m) + сортировка мостов
- **Пространственная сложность**: O(n + m) — список смежности + массивы

## Резюме

Алгоритм находит все мосты за один проход DFS. Ключевое условие `low[v] > tin[u]` проверяет, существует ли путь из поддерева `v` к предкам `u` без использования ребра `(u, v)`. Если такого пути нет — ребро является мостом.
