# Кратчайший путь в невзвешенном графе (BFS)

## Условие задачи
Дан неориентированный невзвешенный граф. Найти длину кратчайшего пути между двумя вершинами (количество рёбер).

## Алгоритм
Используется **поиск в ширину (BFS)** — стандартный алгоритм для кратчайших путей в невзвешенных графах.

### Идея алгоритма
BFS обходит вершины слоями: сначала все вершины на расстоянии 1, затем на расстоянии 2, и т.д. Первая встреча конечной вершины гарантирует кратчайший путь.

### Структуры данных
- `adj[]` — списки смежности
- `dist[]` — расстояние от начальной вершины (-1 = не посещена)
- `queue` — очередь для BFS

### Ход алгоритма
1. **Инициализация**:
   - `dist[i] = -1` для всех вершин
   - `dist[start] = 0`
   - Добавляем `start` в очередь

2. **BFS**:
   - Пока очередь не пуста:
     - Извлекаем вершину `u`
     - Если `u == end` → нашли путь
     - Для каждого соседа `v`:
       - Если `dist[v] == -1` (не посещена):
         - `dist[v] = dist[u] + 1`
         - Добавляем `v` в очередь

3. **Вывод**: `dist[end]` (-1 если недостижима)

### Пример работы
```
Вход:
4 4
1-2, 2-3, 3-4, 1-4
start=1, end=4

BFS:
queue: [1]
dist: [0, -1, -1, -1]

→ u=1, dist[1]=0
  соседи: 2, 4
  dist[2]=1, queue: [2]
  dist[4]=1, queue: [2, 4]

→ u=2, dist[2]=1
  соседи: 1(vis), 3
  dist[3]=2, queue: [4, 3]

→ u=4, dist[4]=1
  4 == end → нашли!

Ответ: 1 (путь 1→4)
```

### Сложность
- **Время**: O(n + m) — каждая вершина и ребро обрабатываются один раз
- **Память**: O(n) — массив расстояний и очередь

### Ключевые моменты реализации
1. **Проверка конца**: можно остановиться сразу при извлечении `end` из очереди
2. **-1 для недостижимых**: инициализируем расстояния как -1
3. **Очередь**: гарантирует обработку вершин в порядке возрастания расстояния
