# Наименьший общий предок (LCA) с бинарным подъёмом

## Условие задачи
Дано дерево (ориентированное от корня). Для множества запросов (u, v) найти наименьшего общего предка (LCA) — ближайшую к корню вершину, которая является предком и u, и v.

## Алгоритм
Используется **бинарный подъём** (Binary Lifting) — предварительная обработка за O(n log n) и ответ на запрос за O(log n).

### Идея алгоритма
**Предварительная обработка**:
- Для каждой вершины u и степени 2^i вычисляем `up[u][i]` — 2^i-го предка u
- `up[u][0]` — непосредственный родитель
- `up[u][i] = up[up[u][i-1]][i-1]` — 2^i-й предок = 2^(i-1)-й предок от 2^(i-1)-го предка

**Ответ на запрос LCA(u, v)**:
1. Выравниваем глубину: поднимаем более глубокую вершину на уровень другой
2. Если вершины совпали — это LCA
3. Иначе поднимаем обе вершины, пока их предки не совпадут
4. LCA — родитель обеих вершин

### Структуры данных
- `adj[]` — списки смежности
- `up[][]` — таблица 2^i-х предков
- `depth[]` — глубина вершины (расстояние от корня)
- `LOG` — максимальная степень двойки (log₂(n))

### Ход алгоритма
**Предобработка (DFS)**:
1. `depth[root] = 0`, `up[root][0] = root`
2. Для каждой вершины u:
   - `up[u][0] = parent`
   - Для i от 1 до LOG-1:
     - `up[u][i] = up[up[u][i-1]][i-1]`
   - Рекурсивно обрабатываем детей

**Запрос LCA(u, v)**:
1. Если `depth[u] < depth[v]`, меняем местами
2. Поднимаем u на уровень v:
   - Для i от 0 до LOG-1:
     - Если `(depth[u] - depth[v]) & (1 << i)`: `u = up[u][i]`
3. Если `u == v` → возвращаем u
4. Поднимаем обе вершины:
   - Для i от LOG-1 до 0:
     - Если `up[u][i] != up[v][i]`:
       - `u = up[u][i]`, `v = up[v][i]`
5. Возвращаем `up[u][0]`

### Пример работы
```
Вход:
5 3
0 (root=1)
1 (parent of 2)
1 (parent of 3)
2 (parent of 4)
2 (parent of 5)

Дерево:
      1
     / \
    2   3
   / \
  4   5

Предобработка:
depth: [0, 1, 1, 2, 2]
up[4]: [2, 1, 1, ...]
up[5]: [2, 1, 1, ...]
up[3]: [1, 1, 1, ...]

Запрос LCA(4, 5):
depth[4]=2, depth[5]=2 (равны)
4 ≠ 5
up[4][0]=2, up[5][0]=2 (равны) → не поднимаем
LCA = up[4][0] = 2

Запрос LCA(4, 3):
depth[4]=2, depth[3]=1
Поднимаем 4: up[4][0]=2, depth=1
2 ≠ 3
up[2][0]=1, up[3][0]=1 (равны) → не поднимаем
LCA = up[2][0] = 1

Ответ:
2
1
1
```

### Сложность
- **Предобработка**: O(n log n) — таблица 2^i-х предков
- **Запрос**: O(log n) — подъём по степеням двойки
- **Память**: O(n log n) — таблица up

### Ключевые моменты реализации
1. **Бинарное представление**: любое число можно представить как сумму степеней двойки
2. **Выравнивание глубины**: сначала делаем вершины на одном уровне
3. **Обратный порядок**: при подъёме идём от больших степеней к меньшим
4. **Корень как свой предок**: `up[root][i] = root` для всех i
