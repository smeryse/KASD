# Кратчайший путь во взвешенном графе (Дейкстра)

## Условие задачи
Дан неориентированный взвешенный граф. Найти длину кратчайшего пути между двумя вершинами (сумму весов рёбер).

## Алгоритм
Используется **алгоритм Дейкстры** с очередью с приоритетом.

### Идея алгоритма
Алгоритм Дейкстры жадно выбирает вершину с минимальным известным расстоянием и релаксирует (улучшает) расстояния до её соседей. Очередь с приоритетом обеспечивает эффективный выбор минимальной вершины.

### Структуры данных
- `adj[]` — списки смежности, хранят пары (сосед, вес)
- `dist[]` — текущее кратчайшее расстояние
- `pq` — очередь с приоритетом (мин-куча), хранит (вершина, приоритет=расстояние)
- `INF` — большое число (long.MaxValue / 2 для избежания переполнения)

### Ход алгоритма
1. **Инициализация**:
   - `dist[i] = INF` для всех вершин
   - `dist[start] = 0`
   - Добавляем `(start, 0)` в очередь

2. **Основной цикл**:
   - Пока очередь не пуста:
     - Извлекаем вершину `u` с минимальным приоритетом
     - Если `u == end` → нашли путь
     - Для каждого соседа `(v, w)`:
       - Если `dist[u] + w < dist[v]`:
         - `dist[v] = dist[u] + w`
         - Добавляем `(v, dist[v])` в очередь

3. **Вывод**: `dist[end]` или -1 если недостижима

### Пример работы
```
Вход:
4 4
1-2 (1), 2-3 (2), 3-4 (3), 1-4 (10)
start=1, end=4

Инициализация:
dist: [0, INF, INF, INF]
pq: [(1, 0)]

→ u=1, dist[1]=0
  соседи: 2(w=1), 4(w=10)
  dist[2]=1, pq: [(2,1)]
  dist[4]=10, pq: [(2,1), (4,10)]

→ u=2, dist[2]=1
  соседи: 1(0+1≠1), 3(w=2)
  dist[3]=1+2=3, pq: [(3,3), (4,10)]

→ u=3, dist[3]=3
  соседи: 2(1+2≠3), 4(w=3)
  dist[4]=min(10, 3+3)=6, pq: [(4,6), (4,10)]

→ u=4, dist[4]=6
  4 == end → нашли!

Ответ: 6 (путь 1→2→3→4)
```

### Сложность
- **Время**: O((n + m) log n) — каждая вершина и ребро обрабатываются с логарифмической стоимостью кучи
- **Память**: O(n + m) — граф, массив расстояний, очередь

### Ключевые моменты реализации
1. **Непрерывная релаксация**: можем добавить одну вершину в очередь несколько раз с разными расстояниями
2. **Проверка актуальности**: в данной реализации останавливаемся при извлечении `end`
3. **INF**: используем long.MaxValue / 2 для безопасного сложения
4. **Взвешенные рёбра**: в отличие от BFS, используем приоритетную очередь
