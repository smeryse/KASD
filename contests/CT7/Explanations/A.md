# A. Топологическая сортировка

## Введение

Задача на построение **топологической сортировки** ориентированного графа. Топологическая сортировка — это линейное упорядочивание вершин, при котором для каждого направленного ребра `(u, v)` вершина `u` предшествует вершине `v`. Если граф содержит цикл, топологическая сортировка невозможна.

Решение использует **поиск в глубину (DFS)** с отслеживанием состояний вершин.

## Идея

Алгоритм основан на свойстве DFS: если мы завершаем обход вершины (выходим из рекурсии), значит, все достижимые из неё вершины уже обработаны. Это означает, что текущую вершину можно добавить в результат **после** всех её потомков.

Используем три состояния вершин:
- **0 (белая)** — вершина ещё не посещена
- **1 (серая)** — вершина находится в процессе обработки (в стеке рекурсии)
- **2 (чёрная)** — вершина полностью обработана

Если при обходе мы встречаем **серую** вершину, значит, в графе есть цикл.

## Разбор кода

### Инициализация

```csharp
adj = new List<int>[n + 1];
for (int i = 1; i <= n; i++)
    adj[i] = new List<int>();
```

Создаём список смежности для графа. Индексация с 1, поэтому массив размера `n + 1`.

```csharp
state = new int[n + 1];
result = new List<int>();
hasCycle = false;
```

- `state` — массив состояний вершин (0/1/2)
- `result` — список для хранения топологической сортировки
- `hasCycle` — флаг обнаружения цикла

### Логика обхода

```csharp
for (int i = 1; i <= n; i++)
{
    if (state[i] == 0)
    {
        DFS(i);
        if (hasCycle)
        {
            Console.WriteLine("-1");
            return;
        }
    }
}
```

Запускаем DFS из каждой непосещённой вершины. Это необходимо, так как граф может быть **несвязным**. Если обнаружен цикл — выводим `-1`.

### Функция DFS

```csharp
private static void DFS(int u)
{
    state[u] = 1;  // Помечаем как обрабатываемую

    foreach (int v in adj[u])
    {
        if (state[v] == 1)  // Нашли цикл!
        {
            hasCycle = true;
            return;
        }
        if (state[v] == 0)  // Непосещённая вершина
        {
            DFS(v);
            if (hasCycle)
                return;
        }
    }

    state[u] = 2;  // Обработка завершена
    result.Add(u); // Добавляем в результат
}
```

Ключевой момент: вершина добавляется в `result` **после** обработки всех её потомков. Это гарантирует, что все рёбра будут направлены от более ранних вершин к более поздним.

### Вывод результата

```csharp
result.Reverse();
Console.WriteLine(string.Join(" ", result));
```

Разворачиваем список, так как вершины добавлялись в порядке **завершения** обработки (обратном топологическому).

## Оптимизации

- **Однопроходный DFS**: не требуется дополнительная память для стека, кроме рекурсии
- **Ранний выход**: при обнаружении цикла немедленно прекращаем работу
- **Временная сложность**: O(n + m) — каждая вершина и ребро обрабатываются один раз
- **Пространственная сложность**: O(n + m) — список смежности + массив состояний

## Резюме

Алгоритм строит топологическую сортировку за один проход DFS. Вершины добавляются в результат в порядке, обратном времени завершения их обработки. Обнаружение цикла происходит через проверку состояния `state[v] == 1` (серая вершина). Финальный разворот списка даёт корректную топологическую сортировку.
