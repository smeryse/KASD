# Топологическая сортировка

## Условие задачи
Дан ориентированный невзвешенный граф. Необходимо построить его топологическую сортировку — такой порядок вершин, при котором все рёбра идут слева направо. Если граф содержит цикл, топологической сортировки не существует.

## Алгоритм
Используется алгоритм на основе **поиска в глубину (DFS)** с отслеживанием состояния вершин.

### Идея алгоритма
Топологическая сортировка возможна только для **ориентированных ациклических графов (DAG)**. Ключевая идея: если мы завершили обход всех потомков вершины в DFS, то эту вершину можно добавить в результат. После обработки всех вершин результат разворачивается.

### Структуры данных
- `adj[]` — массив списков смежности для представления графа
- `state[]` — массив состояний вершин:
  - `0` — не посещена (unvisited)
  - `1` — в процессе обхода (visiting) — мы сейчас в этой вершине или в её потомках
  - `2` — полностью обработана (visited)
- `result` — список для хранения результата сортировки
- `hasCycle` — флаг обнаружения цикла

### Ход алгоритма
1. **Инициализация**: читаем n (вершины) и m (рёбра), строим граф
2. **DFS для каждой непосещённой вершины**:
   - Помечаем вершину как `visiting` (state = 1)
   - Для каждого соседа:
     - Если сосед в состоянии `visiting` → нашли обратное ребро → **цикл** → выводим -1
     - Если сосед не посещён → рекурсивно вызываем DFS
   - После обработки всех соседей помечаем вершину как `visited` (state = 2) и добавляем в результат
3. **Разворот результата**: так как вершины добавляются после обработки потомков, разворачиваем список
4. **Вывод**: печатаем вершины в порядке топологической сортировки

### Пример работы
```
Вход:
6 6
1 2
3 2
4 2
2 5
6 5
4 6

Граф:
4 → 6 → 5
↓   ↓
1 → 2 → 5
↓
3 → 2

DFS(1): 1 → 2 → 5, result = [5, 2, 1]
DFS(3): 3 → 2 (visited), result = [5, 2, 1, 3]
DFS(4): 4 → 6 → 5 (visited), result = [5, 2, 1, 3, 6, 4]

Разворот: [4, 6, 3, 1, 2, 5] ✓
```

### Сложность
- **Время**: O(n + m) — каждая вершина и ребро обрабатываются один раз
- **Память**: O(n + m) — граф и массивы состояний

### Ключевые моменты реализации
1. **Обнаружение цикла**: если встречаем вершину в состоянии `visiting`, значит есть обратное ребро
2. **Порядок добавления**: вершина добавляется в результат *после* обработки всех её потомков
3. **Несколько компонент связности**: запускаем DFS для каждой непосещённой вершины
