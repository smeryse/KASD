# Конденсация графа

## Условие задачи
Конденсация графа G — граф H, где каждая вершина соответствует компоненте сильной связности (КСВ) в G. Ребро в H существует, если есть хотя бы одно ребро из соответствующей КСВ в другую. Требуется найти количество рёбер в конденсации.

## Алгоритм
Используется **алгоритм Косарайю** для нахождения компонент сильной связности.

### Идея алгоритма
Алгоритм Косарайю основан на свойстве: если отсортировать вершины по времени выхода в DFS и запустить DFS в транспонированном графе, то каждый запуск DFS выделит одну компоненту сильной связности.

### Структуры данных
- `adj[]` — списки смежности исходного графа
- `adjRev[]` — списки смежности транспонированного графа (обратные рёбра)
- `visited[]` — посещённые вершины
- `order` — порядок выхода из вершин в первом DFS
- `component[]` — номер компоненты для каждой вершины
- `condensationEdges` — множество рёбер конденсации (HashSet для уникальности)

### Ход алгоритма
**Первый проход (прямой граф)**:
1. DFS по всем непосещённым вершинам
2. После обработки всех потомков добавляем вершину в `order`

**Второй проход (транспонированный граф)**:
1. Разворачиваем `order` (вершины с большим временем выхода — первые)
2. Для каждой вершины в порядке:
   - Если не имеет компоненты → DFS в `adjRev`, присваиваем `componentCount++`

**Подсчёт рёбер конденсации**:
1. Для каждого ребра (u, v) в исходном графе:
   - Если `component[u] != component[v]` → добавляем `(component[u], component[v])` в множество
2. Размер множества — ответ

### Пример работы
```
Вход:
4 4
2→1, 3→2, 2→3, 4→3

Граф:
4 → 3 ↔ 2 → 1

КСВ:
- Компонента 1: {2, 3} (цикл 2↔3)
- Компонента 2: {1}
- Компонента 3: {4}

Конденсация:
4(3) → 3(1) → 1(2)
         ↓
       2(1)

Рёбра конденсации:
- 3→1 (из 4→3)
- 1→2 (из 2→1)

Ответ: 2 ребра
```

### Сложность
- **Время**: O(n + m) — два прохода DFS + проход по рёбрам
- **Память**: O(n + m) — два графа, массивы, множество

### Ключевые моменты реализации
1. **Транспонированный граф**: строим `adjRev` для второго прохода
2. **Порядок обработки**: вершины с большим временем выхода обрабатываются первыми
3. **Уникальность рёбер**: HashSet автоматически удаляет кратные рёбра
4. **Игнорирование петель**: петли не влияют на конденсацию
