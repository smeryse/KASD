# C. Точки сочленения

## Введение

Задача на нахождение всех **точек сочленения** (шарниров) в неориентированном графе. Точка сочленения — это вершина, удаление которой увеличивает количество компонент связности графа.

Решение использует модификацию **поиска в глубину (DFS)** с вычислением времён входа `tin` и точек возврата `low`.

## Идея

Алгоритм похож на поиск мостов, но с важными отличиями:

1. **Для корня DFS**: вершина является точкой сочленения, если у неё более одного ребёнка в дереве DFS
2. **Для остальных вершин**: вершина `u` — точка сочленения, если существует ребёнок `v`, для которого `low[v] >= tin[u]`

Неравенство `low[v] >= tin[u]` означает, что из поддерева `v` нельзя достичь предка `u` без прохождения через саму `u`.

## Разбор кода

### Инициализация

```csharp
adj = new List<int>[n + 1];
for (int i = 1; i <= n; i++)
    adj[i] = new List<int>();
```

Список смежности для неориентированного графа.

```csharp
tin = new int[n + 1];
low = new int[n + 1];
articulationPoints = new HashSet<int>();
timer = 0;
```

- `tin` и `low` — массивы времён входа и возврата
- `articulationPoints` — множество для хранения точек сочленения (автоматически избегает дубликатов)
- `timer` — счётчик времени

### Обход графа

```csharp
for (int i = 1; i <= n; i++)
{
    if (tin[i] == 0)
        DFS(i, -1);
}
```

Запускаем DFS из каждой непосещённой вершины (граф может быть несвязным).

### Функция DFS

```csharp
private static void DFS(int u, int parent)
{
    tin[u] = low[u] = ++timer;
    int children = 0;
```

Инициализируем времена и счётчик детей в дереве DFS.

```csharp
    foreach (int v in adj[u])
    {
        if (v == parent)
            continue;
```

Пропускаем родителя (ребро, по которому пришли).

```csharp
        if (tin[v] != 0)
        {
            low[u] = Math.Min(low[u], tin[v]);
        }
```

Если `v` уже посещена — это обратное ребро. Обновляем `low[u]` через `tin[v]`.

```csharp
        else
        {
            children++;
            DFS(v, u);
            low[u] = Math.Min(low[u], low[v]);
            if (parent != -1 && low[v] >= tin[u])
                articulationPoints.Add(u);
        }
    }
```

Для непосещённой вершины:
1. Увеличиваем счётчик детей
2. Рекурсивно вызываем DFS
3. Обновляем `low[u]` через `low[v]`
4. Проверяем условие точки сочленения: `low[v] >= tin[u]` (но не для корня)

```csharp
    if (parent == -1 && children > 1)
        articulationPoints.Add(u);
}
```

Отдельная проверка для корня: если детей больше одного, корень — точка сочленения.

### Вывод результата

```csharp
var result = new List<int>(articulationPoints);
result.Sort();
Console.WriteLine(result.Count);
Console.WriteLine(string.Join(" ", result));
```

Конвертируем множество в список, сортируем и выводим.

## Оптимизации

- **HashSet для точек сочленения**: автоматическое избегание дубликатов
- **Однопроходный DFS**: точки сочленения находятся во время обхода
- **Временная сложность**: O(n + m + k log k), где k — количество точек сочленения
- **Пространственная сложность**: O(n + m)

## Резюме

Алгоритм находит все точки сочленения за один проход DFS. Ключевое различие с мостами: для точки сочленения проверяется `low[v] >= tin[u]` (включая равенство), а для моста — `low[v] > tin[u]` (строгое неравенство). Корень обрабатывается отдельно: он является точкой сочленения только при наличии более одного ребёнка в дереве DFS.
