# Точки сочленения

## Условие задачи
Дан неориентированный граф. Требуется найти все **точки сочленения** (шарниры) — вершины, удаление которых увеличивает количество компонент связности графа.

## Алгоритм
Используется модифицированный **поиск в глубину (DFS)** с вычислением tin и low.

### Идея алгоритма
Вершина u является точкой сочленения в двух случаях:
1. **u — корень дерева DFS** и имеет более одного ребёнка в дереве DFS
2. **u — не корень** и существует ребёнок v такой, что **low[v] ≥ tin[u]** (нет обратного ребра из поддерева v выше u)

### Структуры данных
- `adj[]` — списки смежности
- `tin[]` — время захода в вершину
- `low[]` — минимальное достижимое время захода
- `articulationPoints` — множество точек сочленения (HashSet для уникальности)
- `timer` — счётчик времени

### Ход алгоритма
1. **Инициализация**: читаем граф
2. **DFS от каждой непосещённой вершины**:
   - Устанавливаем `tin[u] = low[u] = ++timer`
   - Считаем количество детей `children` в дереве DFS
   - Для каждого соседа v:
     - Если v — родитель → пропускаем
     - Если v уже посещена → `low[u] = min(low[u], tin[v])`
     - Если v не посещена:
       - `children++`
       - DFS(v)
       - `low[u] = min(low[u], low[v])`
       - Если `parent != -1 && low[v] >= tin[u]` → u — точка сочленения
   - Если `parent == -1 && children > 1` → u — точка сочленения (корень с ≥2 детьми)
3. **Вывод**: сортируем точки сочленения, выводим количество и номера

### Пример работы
```
Вход:
6 7
1-2, 2-3, 2-4, 2-5, 4-5, 1-3, 3-6

Граф:
    1
   / \
  2---3
 / \   \
4---5   6

DFS(1):
tin[1]=1, low[1]=1, children=0
→ DFS(2): tin[2]=2, low[2]=2
  → DFS(3): tin[3]=3, low[3]=3
    → 1 (обратное): low[3]=min(3,1)=1
    → DFS(6): tin[6]=4, low[6]=4
      → low[6]=4 > tin[3]=3? Да → 3 — ТОЧКА СОЧЛЕНЕНИЯ
    → low[3]=min(1,4)=1
  → low[2]=min(2,1)=1
  → DFS(4): tin[4]=5, low[4]=5
    → DFS(5): tin[5]=6, low[5]=6
      → 2 (обратное): low[5]=min(6,2)=2
      → 4 (обратное): low[5]=min(2,5)=2
    → low[4]=min(5,2)=2
  → low[2]=min(1,2)=1
→ children=1, корень не точка сочленения

Проверка для 2:
low[4]=2 >= tin[2]=2? Да → 2 — ТОЧКА СОЧЛЕНЕНИЯ

Ответ: 2 точки сочленения: 2, 3
```

### Сложность
- **Время**: O(n + m) — один проход DFS
- **Память**: O(n) — массивы и множество

### Ключевые моменты реализации
1. **Корень — особый случай**: корень является точкой сочленения только при children > 1
2. **Неравенство ≥**: важно использовать `>=`, а не `>` (в отличие от мостов)
3. **HashSet**: используем множество для избежания дубликатов
4. **Сортировка вывода**: точки сочленения выводятся в возрастающем порядке
