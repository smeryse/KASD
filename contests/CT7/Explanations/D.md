# Компоненты рёберной двусвязности

## Условие задачи
Компонента рёберной двусвязности — максимальное множество вершин, где для любых двух вершин существует не менее двух рёберно не пересекающихся путей между ними. Требуется разбить граф на такие компоненты.

## Алгоритм
Двухпроходный алгоритм:
1. **Найти все мосты** (как в задаче B)
2. **Пройти по графу, не переходя через мосты**, присваивая номера компонент

### Идея алгоритма
Рёбра, не являющиеся мостами, соединяют вершины внутри одной компоненты двусвязности. Мосты — это рёбра между компонентами.

### Структуры данных
- `adj[]` — списки смежности с индексами рёбер
- `tin[]`, `low[]` — для поиска мостов
- `component[]` — номер компоненты для каждой вершины
- `bridgeEdges` — множество индексов мостов
- `componentCount` — количество компонент

### Ход алгоритма
**Первый проход (поиск мостов)**:
1. DFS для вычисления tin и low
2. Если `low[v] > tin[u]` → ребро (u, v) — мост, добавляем в `bridgeEdges`

**Второй проход (раскраска компонент)**:
1. Для каждой непосещённой вершины:
   - `componentCount++`
   - DFS/BFS, не переходя через мосты
   - Всем достижимым вершинам присваиваем `component[u] = componentCount`

### Пример работы
```
Вход:
6 7
1-2, 2-3, 3-1, 1-4, 4-5, 4-6, 5-6

Граф:
  1-----2
  |    /
  |   /
  4--3
 / \
5---6

Мосты: ребро 1-4 (индекс 4)

Компоненты:
- Компонента 1: вершины {1, 2, 3} (цикл 1-2-3)
- Компонента 2: вершины {4, 5, 6} (цикл 4-5-6)

Вывод:
2
1 1 1 2 2 2
```

### Сложность
- **Время**: O(n + m) — два прохода DFS
- **Память**: O(n + m) — граф, массивы, множество мостов

### Ключевые моменты реализации
1. **Два прохода**: сначала находим мосты, потом раскрашиваем компоненты
2. **Игнорирование мостов**: при раскраске не переходим через рёбра из `bridgeEdges`
3. **Несвязный граф**: обрабатываем все компоненты связности
4. **Нумерация**: компоненты нумеруются с 1, вершины выводятся в порядке 1..n
