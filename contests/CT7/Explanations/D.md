# D. Компоненты рёберной двусвязности

## Введение

Задача на выделение **компонент рёберной двусвязности** в неориентированном графе. Компонента рёберной двусвязности — это максимальное множество вершин, между любыми двумя из которых существует как минимум два рёберно-непересекающихся пути.

Решение использует **двухэтапный алгоритм**: сначала находятся все мосты, затем вершины объединяются в компоненты, игнорируя мосты.

## Идея

Ключевое наблюдение: если удалить все мосты из графа, оставшиеся связные компоненты и будут компонентами рёберной двусвязности.

Алгоритм состоит из двух фаз:
1. **Поиск мостов** через DFS с `tin` и `low`
2. **Раскраска компонент** — обход графа без прохождения по мостам

## Разбор кода

### Инициализация

```csharp
adj = new List<(int, int)>[n + 1];
for (int i = 1; i <= n; i++)
    adj[i] = new List<(int, int)>();
```

Список смежности хранит пары `(вершина, индекс_ребра)` для идентификации мостов.

```csharp
tin = new int[n + 1];
low = new int[n + 1];
component = new int[n + 1];
bridgeEdges = new HashSet<int>();
timer = 0;
```

- `tin`, `low` — для поиска мостов
- `component` — номер компоненты для каждой вершины (0 = ещё не назначена)
- `bridgeEdges` — множество индексов мостов

### Фаза 1: Поиск мостов

```csharp
for (int i = 1; i <= n; i++)
{
    if (tin[i] == 0)
        FindBridges(i, -1);
}
```

Запускаем DFS из каждой непосещённой вершины.

```csharp
private static void FindBridges(int u, int parentEdge)
{
    tin[u] = low[u] = ++timer;

    foreach (var (v, edgeIndex) in adj[u])
    {
        if (edgeIndex == parentEdge)
            continue;

        if (tin[v] != 0)
        {
            low[u] = Math.Min(low[u], tin[v]);
        }
        else
        {
            FindBridges(v, edgeIndex);
            low[u] = Math.Min(low[u], low[v]);
            if (low[v] > tin[u])
                bridgeEdges.Add(edgeIndex);
        }
    }
}
```

Стандартный алгоритм поиска мостов. Если `low[v] > tin[u]`, ребро с `edgeIndex` — мост.

### Фаза 2: Раскраска компонент

```csharp
componentCount = 0;
for (int i = 1; i <= n; i++)
{
    if (component[i] == 0)
    {
        componentCount++;
        AssignComponent(i);
    }
}
```

Для каждой ещё не окрашенной вершины запускаем новую компоненту.

```csharp
private static void AssignComponent(int u)
{
    component[u] = componentCount;
    foreach (var (v, edgeIndex) in adj[u])
    {
        if (component[v] == 0 && !bridgeEdges.Contains(edgeIndex))
            AssignComponent(v);
    }
}
```

Рекурсивно обходим всех соседей, **игнорируя мосты**. Все достижимые вершины получают тот же номер компоненты.

### Вывод результата

```csharp
Console.WriteLine(componentCount);
var result = new int[n];
for (int i = 1; i <= n; i++)
    result[i - 1] = component[i];
Console.WriteLine(string.Join(" ", result));
```

Выводим количество компонент и номер компоненты для каждой вершины.

## Оптимизации

- **Двухэтапный подход**: разделение логики упрощает код
- **HashSet для мостов**: быстрая проверка O(1)
- **Временная сложность**: O(n + m) — два обхода графа
- **Пространственная сложность**: O(n + m)

## Резюме

Алгоритм разделяет задачу на две независимые части: поиск мостов и раскраску компонент. Удаление мостов разбивает граф на компоненты рёберной двусвязности, так как внутри компоненты между любыми двумя вершинами существует как минимум два непересекающихся пути.
