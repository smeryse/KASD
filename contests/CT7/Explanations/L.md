# 2-SAT (2-выполнимость)

## Условие задачи
Дана булева формула в конъюнктивной нормальной форме (КНФ), где каждый дизъюнкт содержит ровно 2 литерала. Определить, выполнима ли формула, и если да — найти значения переменных.

## Алгоритм
Используется **сведение к графу сильной связности** с алгоритмом Косарайю.

### Идея алгоритма
**Представление в виде графа**:
- Каждая переменная xᵢ представлена двумя вершинами: xᵢ (истина) и ¬xᵢ (ложь)
- Дизъюнкт (a ∨ b) эквивалентен импликациям: (¬a → b) и (¬b → a)
- Строим граф импликаций

**Критерий выполнимости**:
Формула невыполнима ⇔ существует переменная xᵢ, где xᵢ и ¬xᵢ достижимы друг из друга (лежат в одной компоненте сильной связности).

**Восстановление ответа**:
Если формула выполнима, присваиваем xᵢ = true, если номер компоненты xᵢ > номер компоненты ¬xᵢ (топологический порядок).

### Структуры данных
- `adj[]`, `adjRev[]` — граф импликаций и транспонированный (2n + 1 вершин)
- `visited[]` — для DFS
- `order` — порядок выхода из вершин
- `component[]` — номер компоненты сильной связности
- `assignment[]` — значение переменных (0 = false, 1 = true)

### Кодирование вершин
- Переменная i (true): вершина i
- Переменная i (false): вершина i + n
- ¬(i > 0) = i + n
- ¬(i < 0) = |i|

### Ход алгоритма
1. **Построение графа**:
   - Для каждого дизъюнкта (a ∨ b):
     - Добавляем рёбра ¬a → b и ¬b → a

2. **Алгоритм Косарайю**:
   - Первый проход (прямой граф): DFS, заполняем `order`
   - Второй проход (транспонированный): DFS в обратном порядке, нумеруем компоненты

3. **Проверка выполнимости**:
   - Для каждой переменной i:
     - Если `component[i] == component[i + n]` → "No solution"

4. **Восстановление ответа**:
   - `assignment[i] = (component[i] > component[i + n]) ? 1 : 0`

5. **Вывод**: значения переменных

### Пример работы
```
Вход:
3 3
1 2      (x₁ ∨ x₂)
-1 2     (¬x₁ ∨ x₂)
-2 3     (¬x₂ ∨ x₃)

Граф импликаций (8 вершин для 3 переменных):
(¬x₁→x₂), (¬x₂→x₁)
(x₁→x₂), (¬x₂→¬x₁)
(x₂→x₃), (¬x₃→¬x₂)

Компоненты сильной связности:
КСВ(¬x₁) = 1, КСВ(x₁) = 2
КСВ(¬x₂) = 1, КСВ(x₂) = 3
КСВ(¬x₃) = 2, КСВ(x₃) = 3

Проверка:
x₁: component[1]=2 ≠ component[4]=1 ✓
x₂: component[2]=3 ≠ component[5]=1 ✓
x₃: component[3]=3 ≠ component[6]=2 ✓

Ответ:
x₁: comp[1]=2 > comp[4]=1 → true (1)
x₂: comp[2]=3 > comp[5]=1 → true (1)
x₃: comp[3]=3 > comp[6]=2 → true (1)

Вывод: 1 1 1
```

### Сложность
- **Время**: O(n + m) — алгоритм Косарайю
- **Память**: O(n + m) — граф и массивы

### Ключевые моменты реализации
1. **Две вершины на переменную**: xᵢ и ¬xᵢ кодируются как i и i+n
2. **Импликации**: (a ∨ b) ≡ (¬a → b) ∧ (¬b → a)
3. **Порядок компонент**: большее значение компоненты = более ранний топологический порядок
4. **Проверка противоречия**: переменная и её отрицание не должны быть в одной КСВ
