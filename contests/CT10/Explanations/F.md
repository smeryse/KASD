# Задача F. Дракончики

## Идея

**Паросочетание с ограничениями**: некоторые пары запрещены, некоторые вершины обязательны.

## Модель

- Левая доля = зелёные дракончики (1..m)
- Правая доля = жёлтые дракончики (m+1..m+k)
- Рёбра = разрешённые пары (исключая запрещённые)

## Алгоритм

1. Строим граф без запрещённых рёбер
2. **Приоритет**: сначала сопоставляем дракончиков, которые **не переносят одиночество**
3. Затем пытаемся дополнить до паросочетания размера $n$
4. Проверяем, что все «обязательные» покрыты

## Почему приоритет важен?

Если сначала сопоставить «свободных» дракончиков, они могут занять тех жёлтых, которые нужны «обязательным». Приоритетная обработка гарантирует, что обязательные будут покрыты.

## Проверка

После нахождения паросочетания:
- Если matching < n → `NO`
- Если какой-то «не переносящий одиночество» не покрыт → `NO`
- Иначе → `YES` + пары

## Пример

```
m=2, k=2, n=1
Forbidden: (1, 3)  ← зелёный 1 ≠ жёлтый 3
Alone-forbidden: 2  ← зелёный 2 не переносит одиночество

Граф:
  Зелёный 1 → Жёлтый 4      (3 запрещён)
  Зелёный 2 → Жёлтый 3, 4   (оба разрешены)

Приоритет: зелёный 2 должен быть сопоставлен
  match: 2 → 3 (или 2 → 4)

Ответ: YES, 2 3 (или 2 4)
```
