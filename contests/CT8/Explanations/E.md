# E. Кратчайшие пути

## Введение

Задача на нахождение **кратчайших путей от одной вершины до всех остальных** в ориентированном взвешенном графе, который может содержать:
- Отрицательные веса рёбер
- Циклы отрицательного веса
- Кратные рёбра и петли

Требуется вывести три типа ответов:
- `*` — вершина недостижима
- `-` — существует путь, но нет кратчайшего (вершина достижима из цикла отрицательного веса)
- Число — длина кратчайшего пути

Решение использует **алгоритм Беллмана-Форда** с дополнительной проверкой на достижимость из циклов отрицательного веса.

## Идея

Алгоритм Беллмана-Форда основан на релаксации всех рёбер графа. После `n-1` итераций все кратчайшие пути найдены (если нет циклов отрицательного веса).

Для обнаружения вершин, достижимых из циклов отрицательного веса:
1. Запускаем ещё `n` итераций релаксации
2. Если расстояние продолжает уменьшаться — вершина достижима из отрицательного цикла

## Разбор кода

### Инициализация

```csharp
int n = int.Parse(parts[0]);
int m = int.Parse(parts[1]);
int s = int.Parse(parts[2]);

var edges = new List<(int from, int to, long weight)>();
for (int i = 0; i < m; i++)
{
    line = Console.ReadLine();
    parts = line.Split(' ', StringSplitOptions.RemoveEmptyEntries);
    int u = int.Parse(parts[0]);
    int v = int.Parse(parts[1]);
    long w = long.Parse(parts[2]);
    edges.Add((u, v, w));
}
```

Используем `long` для весов, так как они могут достигать `10^15`.

### Базовая инициализация

```csharp
const long INF = long.MaxValue / 4;
long[] dist = new long[n + 1];
for (int i = 1; i <= n; i++)
    dist[i] = INF;
dist[s] = 0;
```

Расстояние от `s` до самой себя равно 0, до остальных — бесконечность.

### n-1 итераций Беллмана-Форда

```csharp
for (int i = 0; i < n - 1; i++)
{
    bool changed = false;
    foreach (var (u, v, w) in edges)
    {
        if (dist[u] != INF && dist[u] + w < dist[v])
        {
            dist[v] = dist[u] + w;
            changed = true;
        }
    }
    if (!changed)
        break;
}
```

Релаксируем все рёбра `n-1` раз. Оптимизация: если на итерации ничего не изменилось, досрочно завершаем.

### Детекция отрицательных циклов

```csharp
bool[] reachableFromNegCycle = new bool[n + 1];

for (int i = 0; i < n; i++)
{
    foreach (var (u, v, w) in edges)
    {
        if (dist[u] != INF && dist[u] + w < dist[v])
        {
            dist[v] = dist[u] + w;
            reachableFromNegCycle[v] = true;
        }
        if (reachableFromNegCycle[u])
        {
            reachableFromNegCycle[v] = true;
        }
    }
}
```

Дополнительные `n` итераций для распространения метки «достижим из отрицательного цикла»:
1. Если расстояние продолжает уменьшаться — помечаем вершину
2. Если вершина достижима из помеченной — она тоже помечается

### Вывод результата

```csharp
for (int i = 1; i <= n; i++)
{
    if (dist[i] == INF)
        Console.WriteLine("*");
    else if (reachableFromNegCycle[i])
        Console.WriteLine("-");
    else
        Console.WriteLine(dist[i]);
}
```

Три случая:
- `dist[i] == INF` — вершина недостижима (`*`)
- `reachableFromNegCycle[i]` — достижима из отрицательного цикла (`-`)
- Иначе — выводим длину кратчайшего пути

## Оптимизации

- **Досрочное завершение**: если на итерации нет изменений
- **long для весов**: работа с большими числами до `10^15`
- **Массив достижимости**: быстрое распространение метки отрицательного цикла
- **Временная сложность**: O(n × m) — Беллман-Форд
- **Пространственная сложность**: O(n + m)

## Резюме

Алгоритм Беллмана-Форда обрабатывает графы с отрицательными весами и обнаруживает циклы отрицательного веса. Дополнительное распространение метки «достижим из отрицательного цикла» позволяет корректно классифицировать все вершины. Три типа вывода (`*`, `-`, число) покрывают все возможные случаи: недостижимость, отсутствие кратчайшего пути и существование кратчайшего пути.
