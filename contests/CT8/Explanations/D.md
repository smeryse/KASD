# D. Кратчайший путь длины K

## Введение

Задача на нахождение **кратчайших путей из ровно K рёбер** от заданной вершины S до всех остальных в ориентированном взвешенном графе.

Решение использует **динамическое программирование по количеству рёбер** — аналог алгоритма Беллмана-Форда с фиксированным числом итераций.

## Идея

Определим состояние DP:
- `dp[k][v]` — минимальная длина пути из S в v, использующего ровно k рёбер

Базовый случай:
- `dp[0][S] = 0` (путь из 0 рёбер от S до S)
- `dp[0][v] = ∞` для всех `v ≠ S`

Переход:
- `dp[k][v] = min(dp[k-1][u] + w(u,v))` для всех рёбер `(u, v)` с весом `w`

## Разбор кода

### Инициализация

```csharp
int n = int.Parse(parts[0]);
int m = int.Parse(parts[1]);
int k = int.Parse(parts[2]);
int s = int.Parse(parts[3]);

var edges = new List<(int from, int to, int weight)>();
for (int i = 0; i < m; i++)
{
    line = Console.ReadLine();
    parts = line.Split(' ', StringSplitOptions.RemoveEmptyEntries);
    int u = int.Parse(parts[0]);
    int v = int.Parse(parts[1]);
    int w = int.Parse(parts[2]);
    edges.Add((u, v, w));
}
```

Сохраняем все рёбра в список для последующей итерации.

### Базовый случай

```csharp
const long INF = long.MaxValue / 4;
long[] dist = new long[n + 1];
for (int i = 1; i <= n; i++)
    dist[i] = INF;
dist[s] = 0;
```

Инициализируем расстояния для пути из 0 рёбер: только `dist[s] = 0`, остальные — бесконечность.

### K итераций

```csharp
for (int step = 0; step < k; step++)
{
    long[] newDist = new long[n + 1];
    for (int i = 1; i <= n; i++)
        newDist[i] = INF;

    foreach (var (u, v, w) in edges)
    {
        if (dist[u] != INF)
        {
            newDist[v] = Math.Min(newDist[v], dist[u] + w);
        }
    }

    dist = newDist;
}
```

На каждой итерации:
1. Создаём новый массив `newDist` со значениями INF
2. Для каждого ребра `(u, v, w)` пробуем улучшить путь до `v` через `u`
3. Заменяем `dist` на `newDist` для следующей итерации

**Важно**: использование двух массивов (`dist` и `newDist`) гарантирует, что на шаге `k` мы используем пути ровно из `k-1` рёбер, а не меньше.

### Вывод результата

```csharp
for (int i = 1; i <= n; i++)
{
    if (dist[i] == INF)
        Console.WriteLine(-1);
    else
        Console.WriteLine(dist[i]);
}
```

Если расстояние осталось бесконечным, выводим `-1` (пути из ровно K рёбер не существует).

## Оптимизации

- **Два массива**: разделение текущей и предыдущей итераций
- **Проверка на INF**: избегание переполнения при сложении
- **Список рёбер**: итерация только по существующим рёбрам, а не по всем парам
- **Временная сложность**: O(K × m) — K итераций по m рёбрам
- **Пространственная сложность**: O(n + m) — два массива расстояний + список рёбер

## Резюме

Алгоритм последовательно строит кратчайшие пути с увеличивающимся числом рёбер. На каждой итерации рассматриваются все рёбра графа, и для каждой вершины вычисляется минимальное расстояние с использованием ровно текущего количества рёбер. Разделение массивов на «старый» и «новый» гарантирует корректность подсчёта количества рёбер в пути.
