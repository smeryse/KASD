# C. Цикл отрицательного веса

## Введение

Задача на обнаружение **цикла отрицательного веса** в ориентированном взвешенном графе. Если цикл существует, требуется вывести любой из таких циклов.

Решение использует **алгоритм Флойда-Уоршелла** с восстановлением пути для нахождения отрицательного цикла.

## Идея

После выполнения алгоритма Флойда-Уоршелла матрица расстояний `dist[i][j]` содержит кратчайшие пути между всеми парами вершин. Если для некоторой вершины `i` выполняется `dist[i][i] < 0`, значит, существует цикл отрицательного веса, проходящий через эту вершину.

Для восстановления цикла используем матрицу `next[i][j]`, которая хранит следующую вершину на кратчайшем пути от `i` к `j`.

## Разбор кода

### Инициализация

```csharp
int n = int.Parse(line.Trim());
const int INF = 100000;
int[,] dist = new int[n + 1, n + 1];
int[,] next = new int[n + 1, n + 1];
```

- `dist` — матрица расстояний
- `next` — матрица для восстановления пути
- `INF = 100000` — значение отсутствия ребра (по условию)

```csharp
for (int i = 1; i <= n; i++)
{
    line = Console.ReadLine();
    var parts = line.Split(' ', StringSplitOptions.RemoveEmptyEntries);
    for (int j = 1; j <= n; j++)
    {
        dist[i, j] = int.Parse(parts[j - 1]);
        if (dist[i, j] != INF || i == j)
            next[i, j] = j;  // Следующая вершина на пути
        else
            next[i, j] = -1; // Пути нет
    }
}
```

Инициализируем `next[i][j] = j`, если существует ребро `i → j`. Для главной диагонали `next[i][i] = i`.

### Алгоритм Флойда-Уоршелла

```csharp
for (int k = 1; k <= n; k++)
{
    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            if (dist[i, k] != INF && dist[k, j] != INF && dist[i, k] + dist[k, j] < dist[i, j])
            {
                dist[i, j] = dist[i, k] + dist[k, j];
                next[i, j] = next[i, k];  // Обновляем следующую вершину
            }
        }
    }
}
```

При улучшении пути `i → j` через вершину `k`:
- Обновляем расстояние
- Устанавливаем `next[i][j] = next[i][k]` — следующая вершина на пути `i → k`

### Поиск отрицательного цикла

```csharp
for (int i = 1; i <= n; i++)
{
    if (dist[i, i] < 0)
    {
        // Нашли цикл отрицательного веса
        var cycle = new List<int>();
        int curr = i;
        do
        {
            cycle.Add(curr);
            curr = next[curr, i];
        } while (curr != -1 && curr != i && cycle.Count <= n);
```

Если `dist[i][i] < 0`, то существует отрицательный цикл, проходящий через вершину `i`. Восстанавливаем цикл, двигаясь по матрице `next` от `i` к самой себе.

```csharp
        if (curr != i || cycle.Count > n)
        {
            cycle.Clear();
            int v = i;
            for (int step = 0; step < n; step++)
                v = next[v, i];
            
            cycle.Add(v);
            int start = v;
            while (true)
            {
                v = next[v, i];
                if (v == start || v == -1 || cycle.Count > n)
                    break;
                cycle.Add(v);
            }
        }
```

Если цикл не замкнулся корректно, используем альтернативный метод: сначала «проталкиваем» вершину на n шагов вперёд (гарантированно попадаем на цикл), затем восстанавливаем цикл.

### Вывод результата

```csharp
Console.WriteLine("YES");
Console.WriteLine(cycle.Count);
Console.WriteLine(string.Join(" ", cycle));
```

Если цикл найден, выводим «YES», количество вершин и сами вершины цикла.

```csharp
Console.WriteLine("NO");
```

Если после проверки всех вершин цикл не найден, выводим «NO».

## Оптимизации

- **Матрица next**: восстановление пути без дополнительного DFS
- **Ранний выход**: проверка `dist[i][i] < 0` после завершения алгоритма
- **Временная сложность**: O(n³) — алгоритм Флойда-Уоршелла
- **Пространственная сложность**: O(n²) — две матрицы n×n

## Резюме

Алгоритм Флойда-Уоршелла не только находит кратчайшие пути, но и позволяет обнаружить отрицательные циклы через проверку `dist[i][i] < 0`. Матрица `next` обеспечивает восстановление пути без дополнительных обходов. Если цикл существует, он обязательно будет найден, так как алгоритм рассматривает все пары вершин.
