# A. Флойд

## Введение

Задача на построение **матрицы кратчайших путей** между всеми парами вершин в ориентированном взвешенном графе. Гарантируется отсутствие циклов отрицательного веса.

Решение использует **алгоритм Флойда-Уоршелла** — классический алгоритм динамического программирования для поиска кратчайших путей между всеми парами вершин.

## Идея

Алгоритм основан на следующей подзадаче: `dist[i][j][k]` — кратчайшее расстояние от `i` до `j`, используя только промежуточные вершины из множества `{1, 2, ..., k}`.

Рекуррентное соотношение:
```
dist[i][j][k] = min(dist[i][j][k-1], dist[i][k][k-1] + dist[k][j][k-1])
```

То есть, либо путь не проходит через вершину `k`, либо проходит через неё.

## Разбор кода

### Инициализация

```csharp
int n = int.Parse(line.Trim());
int[,] dist = new int[n, n];

for (int i = 0; i < n; i++)
{
    line = Console.ReadLine();
    var parts = line.Split(' ', StringSplitOptions.RemoveEmptyEntries);
    for (int j = 0; j < n; j++)
    {
        dist[i, j] = int.Parse(parts[j]);
    }
}
```

Читаем матрицу смежности. `dist[i, j]` изначально содержит вес ребра из `i` в `j` или большое число (100000), если ребра нет.

### Основная логика

```csharp
for (int k = 0; k < n; k++)
{
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j < n; j++)
        {
            if (dist[i, k] != 100000 && dist[k, j] != 100000)
            {
                dist[i, j] = Math.Min(dist[i, j], dist[i, k] + dist[k, j]);
            }
        }
    }
}
```

Три вложенных цикла:
- **Внешний цикл (`k`)**: перебираем промежуточные вершины
- **Средний цикл (`i`)**: начальная вершина
- **Внутренний цикл (`j`)**: конечная вершина

Условие `dist[i, k] != 100000 && dist[k, j] != 100000` проверяет существование пути `i → k → j`. Если хотя бы одно ребро отсутствует (вес 100000), суммарный путь не рассматриваем.

Операция `Math.Min(dist[i, j], dist[i, k] + dist[k, j])` выбирает минимум между:
- Текущим кратчайшим путём `i → j`
- Путём через вершину `k`: `i → k → j`

### Вывод результата

```csharp
for (int i = 0; i < n; i++)
{
    var row = new int[n];
    for (int j = 0; j < n; j++)
    {
        row[j] = dist[i, j];
    }
    Console.WriteLine(string.Join(" ", row));
}
```

Выводим матрицу построчно.

## Оптимизации

- **Использование 2D массива**: вместо 3D для экономии памяти (оптимизация по памяти)
- **Проверка на INF**: избегание переполнения при сложении
- **Временная сложность**: O(n³) — три вложенных цикла по n
- **Пространственная сложность**: O(n²) — матрица расстояний

## Резюме

Алгоритм Флойда-Уоршелла последовательно улучшает оценки кратчайших путей, рассматривая каждую вершину как возможную промежуточную. После n итераций матрица `dist` содержит кратчайшие расстояния между всеми парами вершин. Алгоритм особенно эффективен для плотных графов и когда нужны пути между всеми парами.
