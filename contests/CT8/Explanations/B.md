# B. Кратчайший путь – 2

## Введение

Задача на нахождение **кратчайших расстояний от одной вершины до всех остальных** в неориентированном взвешенном графе с неотрицательными весами.

Решение использует **алгоритм Дейкстры** с приоритетной очередью (кучей), что обеспечивает эффективную работу на разреженных графах.

## Идея

Алгоритм Дейкстры основан на жадной стратегии:
1. Начинаем с исходной вершины с расстоянием 0
2. На каждом шаге выбираем вершину с минимальным известным расстоянием
3. «Релаксируем» все исходящие рёбра: если найден более короткий путь к соседу, обновляем расстояние
4. Повторяем, пока все вершины не будут обработаны

Ключевое свойство: когда вершина извлекается из приоритетной очереди, её расстояние уже окончательно (при неотрицательных весах).

## Разбор кода

### Инициализация

```csharp
int n = int.Parse(parts[0]);
int m = int.Parse(parts[1]);

var adj = new List<(int to, int weight)>[n + 1];
for (int i = 1; i <= n; i++)
    adj[i] = new List<(int, int)>();
```

Список смежности хранит пары `(вершина, вес)` для каждого ребра.

```csharp
for (int i = 0; i < m; i++)
{
    line = Console.ReadLine();
    parts = line.Split(' ', StringSplitOptions.RemoveEmptyEntries);
    int u = int.Parse(parts[0]);
    int v = int.Parse(parts[1]);
    int w = int.Parse(parts[2]);
    adj[u].Add((v, w));
    adj[v].Add((u, w));  // Неориентированный граф
}
```

Добавляем рёбра в оба направления, так как граф неориентированный.

### Инициализация расстояний

```csharp
const long INF = long.MaxValue / 2;
long[] dist = new long[n + 1];
for (int i = 1; i <= n; i++)
    dist[i] = INF;
dist[1] = 0;
```

- Используем `long` для избежания переполнения при сложении
- `INF` — достаточно большое число, но не максимальное (чтобы `INF + w` не вызвало переполнение)
- Расстояние от вершины 1 до самой себя равно 0

### Приоритетная очередь

```csharp
var pq = new PriorityQueue<int, long>();
pq.Enqueue(1, 0);
```

`PriorityQueue<TElement, TPriority>` извлекает элементы с минимальным приоритетом. В нашем случае приоритет — текущее расстояние до вершины.

### Основной цикл

```csharp
while (pq.Count > 0)
{
    int u = pq.Dequeue();

    foreach (var (v, w) in adj[u])
    {
        if (dist[u] + w < dist[v])
        {
            dist[v] = dist[u] + w;
            pq.Enqueue(v, dist[v]);
        }
    }
}
```

Для каждой извлечённой вершины `u`:
1. Проходим по всем соседям `v` с весом `w`
2. Проверяем условие релаксации: `dist[u] + w < dist[v]`
3. Если путь через `u` короче, обновляем `dist[v]` и добавляем `v` в очередь

Важно: одна вершина может находиться в очереди несколько раз с разными расстояниями. При извлечении обрабатываем только актуальное (минимальное) расстояние.

### Вывод результата

```csharp
var result = new long[n];
for (int i = 1; i <= n; i++)
    result[i - 1] = dist[i];
Console.WriteLine(string.Join(" ", result));
```

Выводим расстояния от вершины 1 до всех вершин.

## Оптимизации

- **PriorityQueue**: O(log n) на добавление/извлечение
- **Релаксация рёбер**: только при улучшении расстояния
- **Временная сложность**: O(m log n) — каждое ребро обрабатывается один раз, каждая операция с кучей O(log n)
- **Пространственная сложность**: O(n + m)

## Резюме

Алгоритм Дейкстры с приоритетной очередью эффективно находит кратчайшие пути от одной вершины до всех остальных. Жадная стратегия гарантирует оптимальность при неотрицательных весах. Использование `long` предотвращает переполнение, а PriorityQueue обеспечивает логарифмическое время операций с кучей.
