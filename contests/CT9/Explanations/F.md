# Наибольшая общая подстрока K строк

Сначала про общую идею, на которой всё держится. Здесь используется бинарный поиск по длине ответа в сочетании с хэшированием строк. Ключевая мысль в том, что если существует общая подстрока длины X, то существует и общая подстрока длины X-1 (любая подстрока длины X-1, являющаяся префиксом подстроки длины X). Это свойство монотонности позволяет применить бинарный поиск: проверяем, есть ли общая подстрока длины mid, и в зависимости от результата двигаем левую или правую границу.

Первый логический блок — ввод данных в Solve. Этот блок решает задачу "прочитать количество строк k и сами строки". Он нужен, потому что без входных данных нечего обрабатывать. Взаимодействие здесь простое: FastScanner читает число k, затем k строк в массив strings.

Второй логический блок — бинарный поиск по длине в Solve. Этот блок решает задачу "найти максимальную длину общей подстроки". Он нужен, потому что перебор всех длин от 0 до |s[0]| был бы неэффективен. Бинарный поиск работает так: left = 0, right = длина первой строки. На каждой итерации mid = (left + right) / 2, вызывается FindCommonSubstring(strings, mid). Если найдена общая подстрока длины mid, запоминаем её как лучший ответ и пытаемся большую длину (left = mid + 1). Иначе пробуем меньшую (right = mid - 1). Взаимодействие: FindCommonSubstring возвращает найденную общую подстроку или null.

Третий логический блок — функция FindCommonSubstring. Этот блок решает задачу "проверить, есть ли общая подстрока заданной длины len во всех строках". Он нужен как функция проверки для бинарного поиска. Как он работает:

1. Если len == 0, возвращаем пустую строку (тривиально существует).
2. Вычисляем все хэши подстрок длины len в первой строке через GetSubstringHashes, получаем множество commonHashes.
3. Для каждой последующей строки вычисляем её хэши и пересекаем с commonHashes через IntersectWith. Если на каком-то этапе commonHashes становится пустым, дальнейшая проверка бессмысленна — возвращаем null.
4. Если после пересечения со всеми строками commonHashes не пусто, берём любой хэш из множества и через GetSubstringByHash восстанавливаем саму подстроку из первой строки.

Взаимодействие: функция использует GetSubstringHashes для получения множеств хэшей и GetSubstringByHash для восстановления строки по хэшу.

Четвёртый логический блок — функция GetSubstringHashes. Этот блок решает задачу "получить множество хэшей всех подстрок заданной длины len в строке s". Он нужен для эффективной проверки наличия подстрок. Как он работает: используется скользящий хэш. Сначала вычисляется BASE^len mod MOD (power). Затем вычисляется хэш первой подстроки s[0..len-1]. После этого для каждой следующей позиции хэш обновляется за O(1): умножаем на BASE, добавляем новый символ, вычитаем уходящий символ, умноженный на power. Все хэши добавляются в HashSet. Взаимодействие: функция возвращает HashSet<long>, который используется для пересечения множеств.

Пятый логический блок — функция GetSubstringByHash. Этот блок решает задачу "по заданному хэшу найти подстроку в строке s". Он нужен, потому что мы храним только хэши, а для ответа нужно вернуть саму строку. Как он работает: аналогично GetSubstringHashes, вычисляется хэш первой подстроки длины len, проверяется на равенство targetHash. Затем скользящим способом вычисляются хэши остальных подстрок, и при совпадении с targetHash возвращается соответствующая подстрока через Substring. Взаимодействие: вызывается из FindCommonSubstring с хэшем из commonHashes.

Шестой логический блок — вывод результата в Solve. Этот блок решает задачу "вывести найденную наибольшую общую подстроку". Он нужен для соответствия формату вывода. Взаимодействие простое: Console.WriteLine выводит best (или пустую строку, если ничего не найдено).

Седьмой логический блок — FastScanner. Этот блок решает задачу "быстрый ввод данных". Он нужен для эффективного чтения. NextInt() читает целое число, NextString() читает строку.

Восьмой логический блок — Program.Main. Его задача — просто запустить решение: вызвать CommonSubstring.Solve(). Он нужен как точка входа приложения C#.

Итоговое резюме. Программа читает k строк и находит их наибольшую общую подстроку. Используется бинарный поиск по длине: для каждой проверяемой длины вычисляются множества хэшей подстрок для каждой строки, затем эти множества пересекаются. Если пересечение не пусто, общая подстрока существует. Сложность O(K * N * log N), где N — максимальная длина строки, K — количество строк.
