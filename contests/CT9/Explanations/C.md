# Z-функция

Сначала про общую идею, на которой всё держится. Здесь реализуется построение Z-функции — ещё одного классического инструмента из алгоритмов на строках. Z-функция z[i] для строки s определяется как длина наибольшего общего префикса строки s и суффикса s[i..n-1]. Иными словами, z[i] показывает, сколько символов совпадает, начиная с позиции i, с началом строки. Обычно z[0] не определяется (или считается равным 0, так как это тривиально). Ключевая мысль в том, что можно вычислить все значения z[i] за линейное время O(n), используя информацию о ранее вычисленных значениях.

Первый логический блок — ввод данных в Solve. Этот блок решает задачу "прочитать строку s". Он нужен, потому что без входной строки нечего обрабатывать. Взаимодействие здесь простое: FastScanner читает строку, дальше она передаётся в ComputeZFunction.

Второй логический блок — функция ComputeZFunction. Этот блок решает задачу "вычислить z[i] для всех i от 1 до n-1 за O(n)". Он нужен, потому что наивный алгоритм (для каждой позиции сравнивать символы с началом строки) работал бы за O(n²). Здесь используется эффективный алгоритм, основанный на поддержании "Z-блока" — отрезка [l, r], который является самым правым отрезком, совпадающим с префиксом строки.

Давайте разберём, как он работает. Идея в том, чтобы использовать уже вычисленные значения для ускорения вычисления текущего. Для позиции i:

1. Если i находится внутри текущего Z-блока (i < r), то мы можем использовать значение z[i-l], где l — начало Z-блока. Это значение показывает, сколько символов совпадает в позиции i-l с началом строки. Но мы не можем просто скопировать это значение — мы ограничены правым краем блока r. Поэтому z[i] = min(r - i, z[i - l]).

2. После этой "стартовой" инициализации мы пытаемся продолжить匹配 вручную: пока символы s[z[i]] и s[i + z[i]] совпадают, увеличиваем z[i].

3. Если новый Z-блок (начинающийся в i) выходит за пределы текущего правого края (i + z[i] > r), обновляем l = i и r = i + z[i].

Почему это работает за линейное время? Ключевое наблюдение: правый край r может только увеличиваться (никогда не уменьшается), и каждый раз, когда мы делаем сравнение s[z[i]] == s[i + z[i]] внутри цикла while, мы либо увеличиваем r, либо останавливаемся на несовпадении. Суммарное количество увеличений r не превышает n, значит общее число операций O(n).

Третий логический блок — вывод результата в Solve. Этот блок решает задачу "вывести значения z[1..n-1] через пробел". Он нужен для соответствия формату вывода (по условию выводятся значения для индексов 2, 3, ..., |s| в 1-индексации, что соответствует z[1..n-1] в 0-индексации). StringBuilder собирает числа через пробел, в конце Console.WriteLine выводит строку.

Четвёртый логический блок — FastScanner. Этот блок решает задачу "быстрый ввод строки". Он нужен для эффективного чтения больших объёмов данных. NextString() читает посимвольно, пропуская ведущие пробелы и читая до следующего пробела.

Пятый логический блок — Program.Main. Его задача — просто запустить решение: вызвать ZFunction.Solve(). Он нужен как точка входа приложения C#.

Итоговое резюме. Программа читает строку и строит Z-функцию за линейное время. Алгоритм поддерживает самый правый Z-блок [l, r] и использует его для ускорения вычислений: для позиции внутри блока берётся минимальное значение из z[i-l] и ограничения по r, затем делается попытка продолжить匹配. Выводятся n-1 чисел — значения z для позиций 1 до n-1.
