# Поиск периода строки

Сначала про общую идею, на которой всё держится. Здесь используется префикс-функция для нахождения минимального периода строки. Период строки s — это такая строка t, что s может быть представлена как конкатенация одной или нескольких копий t. Требуется найти минимальную по длине такую строку. Ключевая мысль в том, что значение префикс-функции в последней позиции π[n-1] содержит информацию о периодичности строки: если n делится на (n - π[n-1]), то минимальный период имеет длину n - π[n-1].

Первый логический блок — ввод данных в Solve. Этот блок решает задачу "прочитать строку s". Он нужен, потому что без входной строки нечего обрабатывать. Взаимодействие здесь простое: FastScanner читает строку, дальше она передаётся в ComputePrefixFunction.

Второй логический блок — функция ComputePrefixFunction. Этот блок решает задачу "вычислить π[i] для всех i от 0 до n-1 за O(n)". Алгоритм такой же, как в задаче B: для каждой позиции i начинаем с j = π[i-1], пытаемся продолжить匹配, при несовпадении откатываемся через π[j-1]. Линейная сложность обеспечивается тем, что суммарное количество откатов не превышает количества увеличений. Взаимодействие: функция возвращает массив π, который используется в Solve для вычисления длины периода.

Третий логический блок — вычисление длины периода в Solve. Этот блок решает задачу "найти минимальную длину периода". Он нужен, потому что это основная цель программы. Как он работает: вычисляется len = n - π[n-1]. Это значение представляет собой минимальную возможную длину периода. Но важно проверить, действительно ли строка состоит из повторений этого периода: для этого проверяется, делится ли n на len без остатка.

Почему это работает? π[n-1] — это длина наибольшего собственного суффикса s, который является префиксом s. Если такой суффикс имеет длину π[n-1], то оставшаяся часть строки (длиной n - π[n-1]) потенциально может быть периодом. Но это гарантированно работает только если n делится на n - π[n-1]. Если не делится, значит строка не является повторением меньшего паттерна, и минимальный период — это вся строка (длина n).

Пример: для строки "abcabcabc" (n=9) префикс-функция в последней позиции даст π[8] = 6 (суффикс "abcabc" совпадает с префиксом "abcabc"). Тогда len = 9 - 6 = 3, и 9 делится на 3, значит ответ 3. Для строки "abacaba" (n=7) π[6] = 0 (нет собственного суффикса-префикса), len = 7 - 0 = 7, и 7 делится на 7, значит ответ 7.

Четвёртый логический блок — вывод результата. Этот блок решает задачу "вывести длину минимального периода". Он нужен для соответствия формату вывода. Взаимодействие простое: Console.WriteLine выводит вычисленное значение len или n.

Пятый логический блок — FastScanner. Этот блок решает задачу "быстрый ввод строки". Он нужен для эффективного чтения больших объёмов данных (до 10^6 символов). NextString() читает посимвольно, пропуская ведущие пробелы и читая до следующего пробела.

Шестой логический блок — Program.Main. Его задача — просто запустить решение: вызвать PeriodSearch.Solve(). Он нужен как точка входа приложения C#.

Итоговое резюме. Программа читает строку s и находит длину минимального периода. Для этого вычисляется префикс-функция, затем len = n - π[n-1]. Если n делится на len, ответ len, иначе ответ n (вся строка — свой собственный период). Сложность O(n) по времени и памяти.
