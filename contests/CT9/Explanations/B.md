# Префикс-функция

Сначала про общую идею, на которой всё держится. Здесь реализуется построение префикс-функции — классического инструмента из алгоритмов на строках. Префикс-функция π[i] для строки s определяется как длина наибольшего собственного суффикса подстроки s[0..i], который одновременно является префиксом этой же подстроки. "Собственный" означает, что суффикс не совпадает со всей подстрокой s[0..i]. Ключевая мысль в том, что можно вычислить все значения π[i] за линейное время O(n), используя уже вычисленные значения для предыдущих позиций.

Первый логический блок — ввод данных в Solve. Этот блок решает задачу "прочитать строку s". Он нужен, потому что без входной строки нечего обрабатывать. Взаимодействие здесь простое: FastScanner читает строку, дальше она используется для построения префикс-функции.

Второй логический блок — основной алгоритм построения префикс-функции. Этот блок решает задачу "вычислить π[i] для всех i от 0 до n-1 за O(n)". Он нужен, потому что наивный алгоритм (для каждой позиции проверять все возможные суффиксы) работал бы за O(n²), что неприемлемо для |s| до 10^6. Здесь используется алгоритм Кнута-Морриса-Пратта.

Давайте разберём, как он работает. Идея в том, чтобы использовать уже вычисленные значения π для ускорения вычисления текущего. Для позиции i мы начинаем с j = π[i-1] — это длина наибольшего префикса-суффикса для предыдущей позиции. Теперь мы пытаемся "продолжить" этот префикс: если s[i] == s[j], то π[i] = j + 1. Если не равно, мы "откатываемся" к меньшему префиксу: j = π[j-1], и пробуем снова. Этот откат может повториться несколько раз, пока j не станет 0 или мы не найдём совпадение.

Почему это работает за линейное время? Ключевое наблюдение: j может увеличиться не более чем на 1 за шаг внешнего цикла (когда s[i] == s[j]), а откатывается j через π[j-1], то есть строго уменьшается. Суммарное количество откатов не может превысить суммарное количество увеличений, а значит общее число операций O(n).

Третий логический блок — вывод результата. Этот блок решает задачу "вывести все значения π[0..n-1] через пробел". Он нужен для соответствия формату вывода. Взаимодействие простое:StringBuilder собирает числа через пробел, в конце Console.WriteLine выводит строку. Важно: последний пробел не добавляется (проверка i < n - 1).

Четвёртый логический блок — FastScanner. Этот блок решает задачу "быстрый ввод строки". Он нужен для эффективного чтения больших объёмов данных. NextString() читает посимвольно, пропуская ведущие пробелы и читая до следующего пробела. Буферизация уменьшает количество системных вызовов.

Пятый логический блок — Program.Main. Его задача — просто запустить решение: вызвать PrefixFunction.Solve(). Он нужен как точка входа приложения C#. Он взаимодействует со всем остальным так: Main только передаёт управление в Solve.

Итоговое резюме. Программа читает строку и строит префикс-функцию за линейное время. Алгоритм использует идею повторного использования ранее вычисленных значений: для позиции i начинаем с π[i-1] и пытаемся продолжить匹配, откатываясь через π при несовпадении. Выводится n чисел — значения π для всех позиций.
