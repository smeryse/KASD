# Поиск подстроки в строке через Z-функцию

Сначала про общую идею, на которой всё держится. Здесь используется Z-функция для поиска всех вхождений одной строки (образца p) в другую (текст t). Ключевая мысль в том, что если построить Z-функцию для комбинированной строки p + '#' + t (где '#' — разделитель, не встречающийся в строках), то позиции, где z[i] равна длине образца p, будут соответствовать началам вхождений p в t. Это позволяет найти все вхождения за линейное время O(|p| + |t|).

Первый логический блок — ввод данных в Solve. Этот блок решает задачу "прочитать образец p и текст t". Он нужен, потому что без входных строк нечего обрабатывать. Взаимодействие здесь простое: FastScanner читает две строки, дальше они используются для построения комбинированной строки и вычисления Z-функции.

Второй логический блок — построение комбинированной строки и вычисление Z-функции. Этот блок решает задачу "подготовить данные для поиска вхождений". Он нужен, чтобы свести задачу поиска к вычислению Z-функции. Комбинированная строка имеет вид p + '#' + t, где '#' — специальный символ-разделитель. Важно, чтобы разделитель не встречался ни в p, ни в t — тогда z[i] никогда не превысит |p| для позиций в части t. Взаимодействие: ComputeZFunction вычисляет Z-функцию для комбинированной строки за линейное время.

Третий логический блок — поиск позиций вхождений. Этот блок решает задачу "найти все позиции, где p входит в t". Он нужен для получения ответа. Как он работает: мы просматриваем Z-значения для позиций, соответствующих части t в комбинированной строке. Эти позиции начинаются с индекса n + 1 (где n = |p|, плюс 1 для разделителя '#'). Если z[i] == n, это значит, что суффикс комбинированной строки, начинающийся в позиции i, совпадает с префиксом p длиной n — то есть p входит в t. Позиция вхождения в t (в 1-индексации) вычисляется как i - n: мы вычитаем длину p и разделитель, чтобы получить индекс в t.

Четвёртый логический блок — вывод результата. Этот блок решает задачу "вывести количество вхождений и их позиции". Он нужен для соответствия формату вывода. Взаимодействие простое: сначала Console.WriteLine выводит количество элементов в списке positions, затем второй Console.WriteLine выводит все позиции через пробел (string.Join).

Пятый логический блок — функция ComputeZFunction. Этот блок решает задачу "вычислить z[i] для всех i за O(n)". Алгоритм такой же, как в задаче C: поддерживается самый правый Z-блок [l, r], для каждой позиции используется уже вычисленное значение с корректным ограничением, затем делается попытка продолжить匹配. Линейная сложность обеспечивается тем, что правый край r может только увеличиваться.

Шестой логический блок — FastScanner. Этот блок решает задачу "быстрый ввод строк". Он нужен для эффективного чтения больших объёмов данных (до 10^6 символов в каждой строке). NextString() читает посимвольно, пропуская ведущие пробелы и читая до следующего пробела.

Седьмой логический блок — Program.Main. Его задача — просто запустить решение: вызвать SubstringSearch.Solve(). Он нужен как точка входа приложения C#.

Итоговое резюме. Программа читает образец p и текст t, строит комбинированную строку p + '#' + t и вычисляет её Z-функцию. Позиции, где z[i] равна |p|, соответствуют вхождениям p в t. Выводится количество вхождений и их позиции в возрастающем порядке (позиции гарантированно идут в порядке возрастания, так как мы просматриваем Z-массив слева направо). Сложность O(|p| + |t|) по времени и памяти.
