# Хэширование строк и сравнение подстрок

Сначала про общую идею, на которой всё держится. Здесь используется хэширование строк для быстрого сравнения подстрок. Ключевая мысль в том, что мы заранее вычисляем хэши всех префиксов строки, а затем можем получить хэш любой подстроки за константное время. Это позволяет отвечать на запросы о равенстве подстрок очень быстро — O(1) на запрос после O(n) предподсчёта.

Первый логический блок — ввод данных в Solve. Этот блок решает задачу "прочитать исходную строку s и количество запросов m". Он нужен, потому что без входных данных нечего обрабатывать. Взаимодействие здесь простое: fs (FastScanner) читает строку и число, дальше эти значения используются для построения хэш-структуры и обработки запросов.

Второй логический блок — предподсчёт префиксных хэшей и степеней основания. Этот блок решает задачу "подготовить данные для быстрого получения хэша любой подстроки". Он нужен, потому что наивное вычисление хэша каждой подстроки "с нуля" заняло бы O(n) на запрос, что при 10^5 запросов дало бы TLE. Здесь используется классическая техника полиномиального хэширования: h[i] хранит хэш префикса s[0..i-1], а powers[i] хранит BASE^i mod MOD. Взаимодействие между этими двумя массивами критически важно: при вычислении хэша подстроки нам нужно "вычесть" вклад префикса, и для этого мы умножаем h[l] на соответствующую степень BASE.

Третий логический блок — функция GetHash. Этот блок решает задачу "получить хэш подстроки s[l..r] за O(1)". Он нужен для обработки запросов. Как он работает с другими частями: GetHash использует заранее подготовленные h и powers. Формула: hash(s[l..r]) = (h[r+1] - h[l] * powers[r-l+1]) mod MOD. Здесь важно понимать, почему именно так: h[r+1] — это хэш s[0..r], а h[l] * powers[r-l+1] — это хэш s[0..l-1], "сдвинутый" на длину подстроки. Вычитание даёт хэш s[l..r]. Важно также обрабатывать отрицательный результат вычитания: если result < 0, добавляем MOD.

Четвёртый логический блок — цикл обработки запросов в Solve. Этот блок решает задачу "для каждого запроса определить, равны ли две подстроки". Он нужен, потому что это основная цель программы. Взаимодействие с другими блоками такое: для каждого запроса читаются координаты a, b, c, d (в 1-индексации), затем проверяется равенство длин (если длины разные — подстроки точно не равны), затем вычисляются хэши через GetHash (с переводом в 0-индексацию: a-1, b-1 и c-1, d-1), и если хэши совпадают — выводим Yes, иначе No.

Пятый логический блок — FastScanner. Этот блок решает задачу "быстрый ввод данных из потока". Он нужен, потому что при больших объёмах ввода (10^5 символов и 10^5 запросов) стандартный Console.ReadLine() может быть медленным. Как он взаимодействует: NextString() читает строку посимвольно, пропуская пробельные символы в начале и читая до следующего пробела. NextInt() аналогично читает целое число. Буферизация (bufferSize = 64KB) уменьшает количество системных вызовов.

Шестой логический блок — Program.Main. Его задача — просто запустить решение: вызвать SubstringCompare.Solve(). Он нужен как точка входа приложения C#. Он взаимодействует со всем остальным так: Main ничего не знает про ввод, хэши и запросы; он только передаёт управление в Solve.

Итоговое резюме. Программа читает строку и m запросов на сравнение подстрок. Для этого она строит префиксные хэши и степени основания, затем каждый запрос обрабатывает за O(1), получая хэши двух подстрок и сравнивая их. Хэш подстроки вычисляется через разность хэшей префиксов с учётом сдвига. Быстрый ввод через FastScanner обеспечивает укладку в ограничение по времени.
